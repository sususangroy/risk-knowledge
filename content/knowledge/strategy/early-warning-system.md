## 一、预警的核心价值：在火灾发生前灭火

催收是在用户已经逾期后才介入；预警是在用户逾期**之前**，通过识别风险信号，提前干预，防止逾期发生。

**预警 vs 催收的本质区别：**

| 维度 | 预警 | 催收 |
|------|------|------|
| 触发时机 | 逾期前（风险信号出现时） | 逾期后 |
| 目标 | 防止逾期发生 | 最大化逾期后的回款 |
| 干预方式 | 温和（关怀触达、降额） | 强制（外呼、法律手段） |
| 用户感知 | 关怀式，不易产生抵触 | 压力式，容易引发对抗 |
| 成本 | 低 | 高 |

**一个逾期的成本分解：**

```
逾期本金损失 + 催收人力成本 + 机会成本（资金占用） + 坏账拨备
```

预防一次逾期，比事后催回一次逾期的成本低得多。这就是预警体系存在的核心商业逻辑。

## 二、风险信号来源

预警信号分为内部信号和外部信号两大类：

### 内部信号（账户内行为变化）

**还款行为信号**

| 信号 | 说明 | 风险等级 |
|------|------|----------|
| 从全额还款变为最低还款 | 现金流开始紧张的典型信号 | 中 |
| 还款金额持续减少 | 还款能力下降 | 中 |
| 还款时间推迟（还款日后才还） | 资金调度开始困难 | 中 |
| 首次出现迟还（逾期 1–3 天后还清） | 早期预警，需持续观察 | 低-中 |

**账户使用信号**

| 信号 | 说明 | 风险等级 |
|------|------|----------|
| 额度利用率连续多月 > 90% | 长期满负荷，几乎无缓冲 | 高 |
| 取现比例突然上升 | 急需现金，流动性危机信号 | 高 |
| 账户突然不活跃 | 用户可能"放弃"该账户 | 中 |
| 联系方式异常（电话停机、地址变更） | 逃跑风险信号 | 高 |

**申请行为信号**

| 信号 | 说明 | 风险等级 |
|------|------|----------|
| 在本机构频繁申请提额 | 资金需求急迫 | 中 |
| 申请新产品被拒后反复尝试 | 资金链出问题 | 高 |

### 外部信号（第三方数据）

**征信更新信号**

| 信号 | 说明 | 风险等级 |
|------|------|----------|
| 他行出现新增逾期 | 跨机构风险传导，最强预警信号之一 | 高 |
| 近 3 个月新增账户 ≥ 3 家 | 多头借贷开始 | 高 |
| 征信查询次数突增 | 急迫的融资需求 | 中-高 |
| 信用卡整体利用率上升 | 全面资金承压 | 中 |

**就业与收入信号**

| 信号 | 说明 | 风险等级 |
|------|------|----------|
| 社保/公积金停缴 | 失业或换工作的强烈信号 | 高 |
| 公积金缴纳基数骤降 | 降薪信号 | 中 |
| 工资代发账户停止到账 | 失业或换工 | 高 |

**宏观/区域信号**

| 信号 | 说明 | 风险等级 |
|------|------|----------|
| 特定行业大规模裁员 | 如科技行业寒冬，互联网用户风险上升 | 中（需批量预警） |
| 特定区域经济恶化 | 某城市经济下行，当地客户批量预警 | 中 |

## 三、预警分级体系

不同强度的信号对应不同的预警级别，处置方式各异：

### 三级预警体系

**一级预警（黄色）：观察期**

- 触发条件：出现 1–2 个中等风险信号
- 处置方式：系统自动标记，提高数据刷新频率，无主动干预
- 目标：积累更多观察数据，避免误触发

**二级预警（橙色）：主动关怀**

- 触发条件：出现 1 个高风险信号，或多个中等信号叠加
- 处置方式：
  - 系统自动发送关怀短信（非催收语气）
  - 停止主动增额
  - 降低新申请额度上限
  - B 卡分更新频率提高至周度
- 目标：温和接触，了解用户情况，提前布防

**三级预警（红色）：主动干预**

- 触发条件：出现多个高风险信号叠加，或单一极高风险信号
- 处置方式：
  - 人工外呼（以关怀/了解名义，非催收）
  - 主动降额至安全水位
  - 冻结新用款申请
  - 进入催前管理（Pre-Collection）队列
- 目标：直接与用户沟通，评估真实情况，最大限度降低风险敞口

## 四、预警触发器设计

### 规则触发器（简单直接）

直接基于单一指标的阈值触发：

```sql
-- 示例：触发二级预警的规则
IF 他行新增逾期 > 0
  OR (近3个月新增账户 >= 3 AND 征信查询次数 >= 5)
  OR (利用率 >= 90% AND 连续月数 >= 3)
THEN 触发二级预警
```

规则触发器的优点是透明、可解释；缺点是覆盖面有限，遗漏多信号组合的场景。

### 模型触发器（精准）

使用 B 卡评分的变化作为触发器：

```
预警触发条件 = 本月B卡分 < 历史最高分 × (1 - 降幅阈值)
```

例如：用户历史最高 B 卡分 720，本月降至 630（降幅 12.5%），若阈值设为 10%，则触发预警。

也可以独立建一个**预警模型**，输出"未来 3 个月内进入 M1+ 的概率"，超过阈值触发预警。

### 组合触发器（最常用）

规则 + 模型组合：
- 任一高风险规则命中 → 直接三级预警
- B 卡分下降幅度达中等阈值 → 二级预警
- 低风险规则 + B 卡分轻微下降 → 一级预警

## 五、预警处置流程

### 标准处置流程

```
风险信号产生
    ↓
数据处理层：汇聚内外部数据，计算触发指标
    ↓
决策引擎：匹配触发规则/模型，确定预警级别
    ↓
执行层（按级别）：
  一级 → 系统标记 + 提高刷新频率
  二级 → 自动短信 + 停止增额 + 降低新申请额度
  三级 → 人工外呼队列 + 主动降额 + 冻结新用款
    ↓
跟踪与复查：
  用户未好转 → 升级处置
  用户情况改善 → 解除预警
```

### 外呼的沟通策略

预警阶段外呼与催收外呼完全不同，应采用关怀式沟通：

- **错误示范**："您好，我们发现您最近存在风险信号，请立即还清欠款。"
- **正确示范**："您好，我是XX客服，关注到您近期账单金额较大，想了解一下您目前的还款是否方便，我们可以提供分期方案。"

关怀式沟通目标是：让用户感受到帮助，而不是压力。在用户还没有逾期时，对抗情绪会严重损害客户关系。

## 六、预警效果评估

### 准确率 vs 覆盖率的平衡

预警体系面临的核心权衡：

- **预警太敏感（阈值过低）**：覆盖率高，但大量误触发（好客户被误预警），浪费资源，损害用户体验
- **预警太迟钝（阈值过高）**：误触发少，但大量真实风险用户被漏掉（漏掉=等于没有预警）

**衡量指标：**

| 指标 | 定义 | 目标 |
|------|------|------|
| 精确率（Precision） | 预警用户中，真实发生逾期的比例 | 越高越好（减少误触发） |
| 召回率（Recall） | 真实逾期用户中，被预警覆盖的比例 | 越高越好（减少漏掉） |
| 提前预警天数 | 预警触发到实际逾期的平均天数 | 越长越好（留更多处置时间） |
| 预警处置后出催率 | 预警处置后，用户避免逾期的比例 | 越高越好 |

### AB Test 验证预警效果

最严格的验证方式：

- 实验组：触发预警条件后，执行处置动作
- 对照组：触发预警条件后，不执行任何干预
- 对比两组在后续 3–6 个月内的逾期率差异

这可以精确量化预警体系本身的价值（排除"这批用户本来就会好转"的干扰）。

## 七、预警与催收的边界

**预警期**（用户未逾期）：
- 以关怀、维系客户关系为主
- 不能强制要求还款（用户没有逾期，没有法律义务）
- 策略以降额、停止增额为主，减少风险敞口

**一旦用户进入 M0（逾期 1 天）**：
- 进入催收流程
- 可以明确要求还款
- 预警信号变成催收的参考信息（如：预警期间未响应 → 催收难度可能更大）

---

## 小结

贷中预警体系是主动风控的核心工具，其价值在于**将干预时机从逾期后提前到逾期前**：

```
风险信号 → 预警分级 → 差异化处置 → 减少逾期发生
    ↓           ↓           ↓
内外部数据  三级体系   规则+模型触发
```

构建好的预警体系，可以在不增加催收成本的前提下，显著降低 M1+ 转化率，是贷中风控投入产出比最高的工作之一。
