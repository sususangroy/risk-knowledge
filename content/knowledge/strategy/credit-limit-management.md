## 一、额度是什么？为什么重要？

信贷额度（Credit Limit）是金融机构愿意借给某个用户的最大金额上限。

**对用户来说**：额度决定了"我能用多少钱"，是用户黏性的重要来源。  
**对机构来说**：额度决定了"最大风险敞口"，是风险管理的核心杠杆之一。

### 额度的双面性

| 视角 | 额度高的好处 | 额度高的坏处 |
|------|------|------|
| 营收 | 用户用款更多，利息收入增加 | 坏账损失金额更大 |
| 用户 | 满足需求，提升满意度和黏性 | 过度借贷风险 |
| 机构 | 资产规模增长 | 资本占用增加，坏账率可能上升 |

**额度管理的核心命题**：给每个用户分配一个既能满足合理需求、又不超过其还款能力的额度。

## 二、初始额度定价（放款时）

### 2.1 额度定价方法

**方法一：基于收入的倍数法（最简单）**

```
初始额度 = 月收入 × 倍数系数（通常 3–10 倍）
```

倍数根据用户风险等级调整：A 卡高分用户倍数高，低分用户倍数低。

这种方法简单直接，但依赖收入数据的准确性（用户可能虚报）。

**方法二：基于 DTI 的测算（更精确）**

DTI（Debt-to-Income Ratio，负债收入比）是评估还款能力的核心指标：

```
DTI = 月均还款额 / 月收入

新增额度的月还款额 ≤ (目标DTI - 当前DTI) × 月收入
```

例如：目标 DTI 上限设为 50%，用户当前 DTI 30%，月收入 1 万，则：
- 可增加还款额 = (50% - 30%) × 10000 = 2000 元/月
- 对应 12 期贷款，可承受的最大额度 ≈ 24000 元

**方法三：基于 A 卡分的分档定价**

按 A 卡分数划分档位，每档对应固定额度：

| A 卡分段 | 风险等级 | 初始额度 |
|---------|---------|----------|
| ≥ 700 | 极低风险 | 5–10 万 |
| 650–699 | 低风险 | 3–5 万 |
| 600–649 | 中风险 | 1–3 万 |
| < 600 | 高风险 | 0.5–1 万（或拒绝） |

**方法四：额度模型（最精细）**

独立建一个回归模型，预测用户的最优额度：

- 目标变量：用户历史实际使用额度的峰值，或最大还款能力
- 特征：收入、征信数据、消费能力代理变量、A卡分
- 输出：连续的推荐额度，再按业务规则取整

### 2.2 初始额度的保守策略

新产品或新客群上线时，建议采用**保守定价**策略：

- 给比模型预测值低 20–30% 的初始额度
- 原因：建模数据不足，不确定性大；保守开始，表现好后再增额
- 后续根据用户实际表现动态调整

## 三、贷中动态调额

放款后，用户风险会随时间变化，额度也应相应调整。动态调额是贷中风控的核心策略工具。

### 3.1 增额（Credit Limit Increase）

**增额的逻辑**：对表现好的用户主动提升额度，增加利息收入，同时提升用户黏性。

**增额的触发条件**（需同时满足）：

1. **B 卡评分持续高位**：近 3–6 个月评分稳定或上升
2. **还款记录良好**：无逾期，且通常全额还款（不是最低还款）
3. **额度利用率适中**：用款率在 40–80% 区间（太低说明额度已够用，太高说明可能资金紧张）
4. **账龄达标**：通常至少还款 6 个月以上，才有足够的行为数据支撑增额决策
5. **外部风险无恶化**：征信无新增逾期，多头借贷无明显增加

**增额幅度的确定**：

```
增额上限 = min(
  基于收入测算的剩余还款能力对应额度,
  当前额度 × 增幅上限（如 50%）,
  产品最大额度上限
)
```

**主动增额 vs 被动增额**

- **主动增额**：机构主动推送给用户，通常给高价值用户，提升黏性
- **被动增额**：用户申请增额，机构审核后批准，等同于一次小型重新审批

### 3.2 降额（Credit Limit Decrease）

降额是贷中风控最重要的主动防守手段。当用户风险上升时，降低额度可以：
- 直接减少最大潜在损失
- 给用户发送明确信号（"你的信用状况需要改善"）

**降额触发信号（按紧急程度排序）**：

| 信号类型 | 具体表现 | 紧急程度 |
|----------|----------|----------|
| 征信外部逾期 | 其他机构出现新增逾期 | 高 |
| B 卡分大幅下降 | 评分连续 2–3 期下降 >30 分 | 高 |
| 多头借贷突增 | 短期内新增多家借贷机构 | 高 |
| 账户利用率持续 100% | 额度长期用满，资金紧张 | 中 |
| 还款方式恶化 | 从全额还款变为最低还款 | 中 |
| 社保/公积金断缴 | 可能失业 | 中 |
| 联系方式异常 | 电话停机、无法联系 | 低（但需监控） |

**降额策略设计**：

| 风险等级 | 降额幅度 | 是否通知用户 | 是否同时降低新申请额度 |
|----------|----------|------|------|
| 轻微风险 | 降至当前使用额度（不影响在用余额） | 否（静默降额） | 是 |
| 中等风险 | 降至当前额度 50% | 是（发送通知） | 是 |
| 高风险 | 冻结账户（降至 0） | 是 | 是，暂停新申请 |

> **注意**：降额不能低于用户当前已使用的余额，否则会人为制造逾期（用户超额了还不知道）。实践中通常设定为"降至不低于当前余额"。

### 3.3 冻结与注销

情节严重时，降额还不够：

**冻结（Freeze）**：暂停账户新增用款，但已有余额正常还款。用于：
- 用户失联
- 出现欺诈风险信号
- 严重外部风险事件（如宏观经济危机时批量冻结高风险账户）

**注销（Close）**：主动关闭账户，通常在：
- 用户长期不活跃（没有用款，保留账户无意义）
- 用户已完全还清且风险极高，不再开展业务

## 四、额度管理的数据监控

### 4.1 关键监控指标

| 指标 | 定义 | 监控频率 |
|------|------|----------|
| 额度利用率 | 已用额度 / 授信总额度 | 月度 |
| 增额率 | 当月增额人数 / 存量用户数 | 月度 |
| 降额率 | 当月降额人数 / 存量用户数 | 月度 |
| 增额后坏账率 | 增额用户中后续逾期比例 | 季度（需留表现期） |
| 降额有效性 | 降额后用户坏账率 vs 未降额对照组 | 季度 |

### 4.2 增额有效性验证

增额是否真的提升了收益，需要通过 AB Test 验证：

- **实验组**：符合增额条件的用户，实际执行增额
- **对照组**：同等条件用户，不增额
- **观察指标**：两组的利息收入、坏账损失、净利润对比

这是唯一能排除"本来就表现好"（自选择偏差）干扰的方法。

## 五、额度管理常见误区

**误区1：增额只看 B 卡分，不看外部数据**

B 卡分是基于内部行为的，用户可能在外部已经开始出问题（多头借贷、外部逾期）。增额前必须结合最新的外部征信数据。

**误区2：降额一定会让用户流失**

适度、有理由的降额（如"您的信用状况需要改善"）有时反而会让用户重视、主动改善。流失更多发生在无故、突然的大幅降额。

**误区3：额度越高，用户价值越高**

不是线性关系。高额度用户如果超出还款能力，坏账损失会侵蚀所有利息收入。最优额度是**用户愿意充分使用且能稳定还款**的额度，而不是技术上能批到的上限。

## 小结

额度管理贯穿信贷产品的完整生命周期：

```
申请时：初始额度定价（A卡分 + 收入测算）
    ↓
放款后：持续监控（B卡分 + 行为数据）
    ↓
条件达成：增额（提升收益）或降额（控制风险）
    ↓
极端情况：冻结 / 注销
```

额度管理的艺术在于：给每个用户在正确的时间点，分配正确的额度。这需要模型、策略、数据监控三位一体的协同工作。
