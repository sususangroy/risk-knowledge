## 业务背景

贷后风险管理的核心目标是**降低逾期损失，提升催收效率**。从 0 到 1 搭建贷后体系，需要先理解入催人群的本质，再逐步构建催收模型和策略框架，最终实现精细化运营。

---

## Part 1. 入催人群分类

入催用户并非同质，按逾期原因可分为五类，对应完全不同的催收策略：

| 类型 | 典型特征 | 催收难度 | 核心策略 |
|------|----------|----------|----------|
| A 型：遗忘型 | 忘记还款日，提醒即还 | 极低 | 自动短信/APP推送 |
| B 型：惯性逾期 | 习惯性晚还几天，有还款能力 | 低 | 短信提醒 + 少量人工 |
| C 型：资金困难 | 短期流动性不足，主观愿意还 | 中 | 协商分期/延期方案 |
| D 型：不想还 | 有还款能力，主观拒绝或逃避 | 高 | 强力催收 + 法律手段 |
| E 型：欺诈型 | 申请时即无还款意图 | 极高 | 停催，转法务或核销 |

> **关键洞察**：M0 阶段以 A/B 型为主，出催率高；M1+ 阶段 C/D/E 型占比上升，催收成本急剧增加。

## Part 2. M0 催收框架：从 0 到 1

### 阶段一：建立 M0 基础模型

**目标**：用模型替代经验，识别哪些 M0 用户更容易出催。

构建 M0 出催预测模型（C卡），特征来源：
- 还款能力：收入、负债、学历
- 还款意愿：历史履约、贷前申贷行为
- 基本信息：A卡分、B卡分

基础模型产出后，可按分数高低分层触达，优先对高分用户使用自动化催收，节省人工资源。

### 阶段二：构建缓催响应模型 + AB Test 验证

**背景**：M0 用户中，大量 A/B 型用户不需要人工干预，自然会还款或响应短信。若对所有人都人工外呼，资源浪费严重。

**缓催响应模型**：预测哪些用户在"缓催"（仅短信/APP通知，不外呼）的情况下会在 N 天内自行出催。

**关键参数确定**：
- **缓催时间窗口 N**：分析出催率随天数的分布曲线，找到出催率趋于平稳的拐点。拐点前为自然出催期，拐点后才需要主动干预。
- **模型 Target**：在缓催窗口 N 天内出催为正样本，N 天后仍未出催为负样本

**AB Test 验证**：
- 实验组：按模型分层，高缓催得分用户仅触达短信
- 对照组：全量人工外呼
- 对比两组出催率与催收成本，评估模型价值

### 阶段三：构建 N 天流转模型

**背景**：缓催期结束后，剩余未出催用户中，仍有部分是高价值的催收对象。需要二次建模，精确识别"强催有效"的用户。

**构建思路**：

1. 以第 N 天（缓催截止点）为观察点，对仍在催收库中的用户建模
2. 排除缓催样本（因为其出催动作已被前一模型解释）
3. 预测目标：从第 N 天到 M0 结束，该用户是否出催

> **为何要分两步而非一个整体 M0 模型？**
> 若直接对全量 M0 用户建模，缓催期内的大量自然出催用户会主导模型，导致模型本质上等同于缓催响应模型，对强催人群的识别能力弱。

### 阶段四：M0 催收策略框架

结合两个模型，M0 阶段形成完整的四分策略框架：

```
入催用户
├── 缓催响应模型高分 → 自动短信触达，观察 N 天
│   ├── N 天内出催 ✓ → 结束
│   └── N 天后未出催 → 进入 N 天流转模型
│       ├── 流转模型高分 → 高优先级人工外呼
│       └── 流转模型低分 → 低频触达 / 保持观察
└── 缓催响应模型低分 → 立即人工外呼
```

## Part 3. M0 催收优化

### 分群精细化

基础 M0 框架建立后，可在以下维度进行分群建模，提升效果：

| 分群维度 | 说明 |
|----------|------|
| 新老入催 | 首次入催 vs 再次入催（后者有历史催记信息） |
| 还款期数（MOB） | 刚放款 vs 还了多期（还款能力验证程度不同） |
| 产品类型 | 不同利率、期限、金额的产品客群差异大 |

### 能力意愿分析接入

将 C 卡的能力分和意愿分接入 M0 策略：

- **高能力低意愿**：强化催收压力（法律提示、升频外呼）
- **低能力高意愿**：提供协商通道（分期/减免方案）
- **低能力低意愿**：尽早评估催收 ROI，及时停催

### 敏感性分析接入

对 M0 用户建立敏感性分析（参见"催收能力意愿分析"），找到各类人群催收动作的边际收益拐点，避免过度催收：
- 高敏感用户：1-2 次外呼即触达，减少无效拨打
- 叛逆型用户：降频处理，换触达方式（短信→邮件→上门）

## Part 4. M1+ 催收框架

### M1+ 阶段特点

| 指标 | M0 | M1+ |
|------|----|-----|
| 出催率 | 高（40–70%+） | 低（5–20%） |
| 单笔损失 | 当期账单 | 所有剩余未还金额 |
| 催收成本 | 低 | 高 |

M1+ 阶段损失敞口大，即使出催率低，识别出那 5–20% 的可出催用户仍然价值巨大。

### M1+ 模型构建

**初期**：样本不足时，将 M1–M3（DPD 31–120）合并为一个整体模型：
- 观察点：用户进入 M1 的起始日
- 表现期：对应阶段结束（如 M1 为第 60 天）
- Target：出催一期即为正样本（完全出催正样本过少，以出催一期宽松定义）
- 必须合案：同一用户多笔在途订单合并，否则正样本严重不足

**成熟期**：样本充足后，分别构建 M1、M2、M3 精细化模型：
- M1 模型重点：识别资金困难但有意愿的用户，推协商方案
- M2/M3 模型重点：识别少数可出催的高价值用户，集中资源

### M1+ 策略框架

结合 M1+ 模型分数和敏感性分析，构建路由策略：

```
M1+ 入催用户
├── C 卡高分（出催概率高）
│   ├── 高敏感 → 常规外呼，快速跟进
│   └── 低敏感 → 上门催收 / 法务函
├── C 卡中分
│   ├── 高能力低意愿 → 强催 + 法律警示
│   └── 低能力高意愿 → 协商分期
└── C 卡低分（出催概率极低）
    └── 停催 or 批量发函，准备核销
```

## Part 5. 指标体系建设

贷后管理需要配套指标体系，持续监控催收效果：

### 核心指标

| 指标 | 定义 | 监控频率 |
|------|------|----------|
| 入催率 | 新增逾期用户数 / 存量用户数 | 日/周 |
| 出催率 | 各 M 阶段出催用户数 / 入催用户数 | 周/月 |
| 催收效率 | 催回金额 / 催收人力成本 | 月 |
| 流转率 | M0→M1 用户占比 | 月 |
| 核销率 | M3+ 未出催用户占比 | 月/季 |

### 模型监控

| 指标 | 预警阈值 | 说明 |
|------|----------|------|
| PSI（人群稳定性）| > 0.2 触发重建 | 入催人群分布变化 |
| KS 值衰减 | 下降 > 15% 预警 | 模型区分度衰退 |
| 高低分出催率差 | 收窄 > 20% 预警 | 模型预测失效信号 |

---

## 总结

贷后风险管理体系的搭建遵循"先有策略框架，再有模型精化"的路径：

1. **理解人群**：A-E 五类人群决定了基本策略方向
2. **建基础模型**：M0 整体 C 卡，验证区分度
3. **拆解流程**：缓催响应模型 + N 天流转模型，形成 M0 策略树
4. **精细运营**：能力意愿四象限 + 敏感性分析，差异化催收
5. **延伸至 M1+**：合案建模，低出催率中识别高价值用户

模型只是工具，催收策略的持续优化才是贷后管理体系的核心竞争力。
