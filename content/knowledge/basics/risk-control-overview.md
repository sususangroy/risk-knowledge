## 一、先从一笔贷款说起

假设你在某 App 上申请了一笔 2 万元的消费贷款，整个过程可能不到 1 分钟就完成了审批。

但在这 1 分钟里，金融机构的风控系统悄悄做了很多事：

1. 验证你的身份不是伪造的
2. 查询你的征信记录
3. 调用十几个外部数据接口
4. 运行反欺诈模型
5. 运行信用评分模型
6. 根据评分决定额度
7. 完成放款

这一套自动化决策流程，就是**贷前风控**的核心。

而在你拿到这笔钱之后，机构不是"放出去就不管了"：
- 每个月系统都会刷新你的风险评分
- 如果你的信用状况恶化，系统会预警甚至降低你的额度
- 如果你逾期了，催收模型会计算最有效的催收方式

**这就是信贷风控的三个阶段：贷前、贷中、贷后。**

## 二、贷前风控：决定"借还是不借"

贷前是风控最重要的阶段，因为**一旦把钱借给了错误的人，后续的补救成本极高**。

### 2.1 反欺诈：排除假人

信用风险和欺诈风险是两种完全不同的风险：

| 维度 | 信用风险 | 欺诈风险 |
|------|----------|----------|
| 本质 | 借款人无力或不愿还款 | 申请人提供虚假信息骗取贷款 |
| 预测方式 | 信用模型（概率预测） | 欺诈识别（异常检测） |
| 处置方式 | 风险定价、额度控制 | 直接拒绝，上报 |

反欺诈的核心任务是：**在评信用之前，先验证申请人的真实性**。

**常见反欺诈手段：**

- **身份验证**：活体检测（防止照片冒充）、身份证 OCR + 人脸比对
- **设备风险识别**：同一设备同时提交多个申请（团伙欺诈信号）
- **网络风险**：同一 IP 来了大量申请，或使用代理 IP
- **行为生物特征**：申请表填写速度异常快（机器人提交）、粘贴行为（不是手动输入）
- **信息一致性核查**：手机号/银行卡/身份证不匹配

> **为什么不能只用信用模型？**
> 欺诈者可能有良好的信用记录（用真实身份骗贷），信用模型会给他高分。反欺诈必须独立于信用风险，作为第一道防线。

### 2.2 硬规则过滤：强制拒绝

通过反欺诈检测后，进入硬规则层。硬规则是**无论分数多高都必须拒绝**的条件：

| 硬规则示例 | 原因 |
|-----------|------|
| 年龄 < 18 岁或 > 70 岁 | 法律约束或还款能力显著下降 |
| 当前有未结清逾期 | 已经还不起了，再借只会更坏 |
| 征信有呆账/核销记录 | 历史严重违约，通常一票否决 |
| 命中司法失信被执行人名单 | 老赖名单，主观不还款 |
| 近 6 个月内有欺诈记录 | 历史欺诈前科 |

**设计硬规则的原则**：宁可少设，不可误杀。硬规则拒绝的客户，永远没有机会被重新评估。如果把条件设得太宽，会误杀大量好客户。

### 2.3 信用评分：连续打分

通过硬规则后，进入申请评分卡（A 卡）打分阶段：

- A 卡根据申请信息和征信数据，输出一个信用分（通常 300–900）
- 分数越高，预测该申请人未来逾期的概率越低
- 按分数区间设置不同策略（自动通过/人工审核/自动拒绝）

评分卡的建模细节请参见《申请评分卡（A卡）详解》。

### 2.4 额度与利率定价

通过审批后，还需要决定"借多少"和"利率多少"：

**额度定价**：根据申请人的还款能力（收入、负债）和风险水平（A 卡分）综合确定。

**风险定价（Risk-Based Pricing）**：高风险客户收更高利率，以覆盖潜在损失：

```
利率 = 资金成本 + 运营成本 + 风险成本 + 利润目标

风险成本 = PD × LGD × EAD / 贷款金额
```

## 三、贷中风控：监控已放款用户的风险

放款不是终点，是风控工作的另一个开始。

### 3.1 行为评分卡（B 卡）：动态更新风险评分

A 卡只在申请时打分一次，而 B 卡是持续运行的：

- 每月（或每季度）根据用户的最新行为数据重新打分
- B 卡的特征来自放款后的行为：还款记录、账户利用率、外部征信更新数据等
- B 卡分持续下降 = 用户风险在上升

B 卡与 A 卡的关系：A 卡决定"要不要借"，B 卡决定"现在风险有多大"。

### 3.2 额度管理：动态调整

基于 B 卡评分，对存量客户实施动态额度策略：

| B 卡信号 | 策略 |
|----------|------|
| 持续高分，还款良好 | 主动增额（提升用户黏性和收益） |
| 评分稳定 | 维持现有额度 |
| 评分下降 | 停止增额，观察 |
| 评分大幅下降 | 主动降额或冻结 |

### 3.3 预警体系：在逾期前主动干预

当风险信号出现时，预警系统触发干预：

**内部信号**：
- 还款方式由全额变为最低还款
- 账户利用率连续多月超过 90%

**外部信号**：
- 他行出现新增逾期
- 近期大量新开信贷账户（多头借贷）
- 社保/公积金断缴（失业信号）

**处置方式**：
- 轻度风险：系统标记，提高数据更新频率
- 中度风险：发送关怀短信，停止增额
- 高度风险：人工外呼，主动降额

预警与催收的本质区别：预警是在用户**还没有逾期时**介入，用关怀的方式，而不是催收的压迫方式。

## 四、贷后风控：逾期后最大化回款

尽管贷前和贷中做了大量工作，仍然会有部分用户逾期。贷后风控的目标是**最大化逾期资产的回收**。

### 4.1 逾期分级

| 逾期级别 | 逾期天数 | 主要特征 |
|----------|----------|----------|
| M0 | 1–30 天 | 多为忘记还款或短期资金周转，出催率高 |
| M1 | 31–60 天 | 风险开始积累，出催率明显下降 |
| M2 | 61–90 天 | 难催，损失加重 |
| M3+ | 90 天以上 | 不良贷款，损失为所有剩余未还金额 |

### 4.2 催收策略

不同逾期阶段使用不同催收手段：

| 阶段 | 主要手段 | 特点 |
|------|----------|------|
| M0 | 短信/APP推送 | 低成本，大量用户只是忘记还款 |
| M0 中后期 | 人工外呼 | 成本升高，针对未自然出催用户 |
| M1 | 密集外呼 | 压力增大 |
| M2+ | 外呼 + 法律提示 | 高强度催收 |
| M3+ | 律师函/诉讼 | 最后手段 |

### 4.3 催收评分卡（C 卡）

催收评分卡预测：**这个逾期用户，被催回款的概率有多大？**

C 卡的意义在于：催收资源有限（外呼人员、法律手段都有成本），C 卡帮助把资源集中在最有可能催回款的用户上，而不是平均分配。

详见《催收评分卡（C卡）构建思路》。

## 五、数据是风控的血液

贷前贷中贷后的每个环节，都依赖数据：

### 数据类型全景

| 数据类型 | 来源 | 主要用途 |
|----------|------|----------|
| 央行征信 | 人民银行征信中心 | 信用历史、逾期记录、负债情况 |
| 商业征信 | 同盾、百融等 | 多头借贷、欺诈黑名单 |
| 社保/公积金 | 政府数据 | 收入验证、就业状态 |
| 银行流水 | 银行数据 | 真实收入和支出 |
| 设备/行为数据 | App 收集 | 反欺诈、行为特征 |
| 运营商数据 | 三大运营商 | 手机在网时长、实名认证 |

### 数据使用的合规边界

- 数据使用需要用户明确授权
- 不能使用与还款能力/意愿无关的敏感数据（如健康数据、宗教信仰）
- 征信数据查询需记录在案，频繁查询本身就是风险信号

## 六、核心指标体系

### 贷前指标

| 指标 | 定义 | 关注方向 |
|------|------|----------|
| 通过率 | 审批通过 / 申请总量 | 太高 = 风控过松；太低 = 损失客户 |
| 首逾率 | 首期即逾期 / 放款总量 | 反映准入质量，越低越好 |
| 欺诈拒绝率 | 反欺诈拒绝 / 申请总量 | 监控欺诈攻击强度 |

### 贷中指标

| 指标 | 定义 | 关注方向 |
|------|------|----------|
| M0 转 M1 率 | 正常 → 逾期的比例 | 反映贷中风险变化 |
| 迁移率 | 各逾期阶段的流转比例 | 领先于坏账率的预警指标 |
| B 卡评分均值变化 | 存量客户 B 卡分趋势 | 整体客群风险变化 |

### 贷后指标

| 指标 | 定义 | 关注方向 |
|------|------|----------|
| M1 逾期率 | M1 余额 / 总贷款余额 | 最常用的资产质量指标 |
| 出催率 | 逾期后恢复正常的比例 | 催收效率 |
| 核销率 | 坏账核销 / 总余额 | 最终损失 |
| 催收效率 | 催回金额 / 催收投入 | 催收 ROI |

## 七、风控的本质：在风险和收益之间找平衡

很多人以为风控就是"拒绝尽可能多的高风险客户"，但这是错误的理解。

**真正的风控目标是最大化风险调整后的收益（RAROC）：**

```
RAROC = (利息收入 - 资金成本 - 运营成本 - 坏账损失) / 资本占用
```

**过严的风控会：**
- 拒绝大量实际上会还款的客户（I类错误）
- 损失本可获得的利息收入
- 让高质量客户转去竞争对手

**过松的风控会：**
- 批准大量实际上会违约的客户（II类错误）
- 坏账侵蚀所有利息收入
- 资本损失，极端情况危及经营

风控人的核心工作，就是不断调整这个平衡点，在可接受的风险水平下，最大化业务规模和收益。
