## 一、为什么要量化风险？

信贷业务的本质是：把钱借出去，赚取利息，同时承担损失的风险。

如果你不知道这笔钱有多大概率收不回来，你就不知道该收多少利息才能覆盖风险成本。**风险量化，就是让这个"可能损失"从模糊的感觉变成可以计算的数字。**

国际银行业通用的风险量化框架（来自《巴塞尔协议》）将信贷损失分解为三个参数：

```
预期损失（EL）= PD × LGD × EAD
```

- **PD**：违约概率（借款人会不会不还）
- **LGD**：违约损失率（如果不还，最终损失多少）
- **EAD**：违约时风险敞口（如果不还，当时欠了多少）

下面逐一解释。

---

## 二、PD：违约概率

### 什么是 PD？

PD（Probability of Default）= 借款人在未来某段时间内发生违约的概率。

**直白来说**：这个人有多大可能性不还钱？

- PD = 0.01 → 1% 的概率违约，很安全
- PD = 0.30 → 30% 的概率违约，风险很高

### PD 的时间维度

PD 必须对应一个时间窗口：
- **1 年 PD**：最常见，未来 12 个月内违约的概率
- **贷款期限 PD**：整个贷款存续期内的违约概率（期限越长，累计 PD 越高）

### PD 是怎么估算的？

**历史违约率法**：统计历史上同类客群的实际违约比例，作为 PD 的估计：

```
例：过去3年，信用分 620–640 分段的客户，
实际违约率为 8%，则该分段的 PD ≈ 8%
```

**评分模型法**：申请评分卡（A 卡）或行为评分卡（B 卡）直接输出违约概率，即每个用户的个性化 PD。

**影响 PD 的主要因素：**

| 因素 | PD 变化方向 |
|------|-------------|
| 有历史逾期记录 | PD 显著上升 |
| 多头借贷（同时向多家借款） | PD 上升 |
| 负债收入比（DTI）高 | PD 上升 |
| 稳定工作、收入证明完整 | PD 下降 |
| 长期客户、还款记录良好 | PD 下降 |

---

## 三、LGD：违约损失率

### 什么是 LGD？

LGD（Loss Given Default）= 发生违约后，最终无法收回的金额占违约时敞口的比例。

换算关系：

```
LGD = 1 - 回收率（Recovery Rate）
```

**直白来说**：如果这个人真的不还钱，最后我们能收回多少？

- LGD = 20% → 违约后能收回 80%，损失小
- LGD = 80% → 违约后只能收回 20%，损失惨重

### 不同产品的 LGD 差异

LGD 受**抵押物**影响最大：

| 产品类型 | 典型 LGD | 原因 |
|----------|---------|------|
| 住房按揭贷款 | 10%–30% | 有房产抵押，可拍卖变现 |
| 汽车贷款 | 20%–40% | 有车辆抵押，但折旧快 |
| 信用卡 | 60%–80% | 无抵押，催收成本高 |
| 无担保消费贷 | 70%–90% | 无抵押，且金额小、诉讼成本高于催回金额 |

> **无抵押消费贷的 LGD 为什么这么高？**
>
> 因为追讨的成本（律师费 + 诉讼费 + 人工）可能比欠款本身还高。对于 1000 元的欠款，打官司追讨可能花掉 2000 元，机构宁可核销，LGD 接近 100%。

### 影响 LGD 的因素

- **有无抵押物**：最重要的因素
- **抵押物的流动性**：房产 > 汽车 > 设备（变现能力不同）
- **借款人经济状况**：连房子都没了的人，比有点存款的人的 LGD 更高
- **催收效率**：好的催收策略可以降低 LGD

---

## 四、EAD：违约时风险敞口

### 什么是 EAD？

EAD（Exposure at Default）= 违约发生时，借款人还欠机构多少钱。

**直白来说**：如果他现在违约，我们有多少钱"在外面"？

### 不同产品的 EAD 计算

**定额贷款（消费贷/房贷）**

EAD = 当前未偿还本金余额

例：借款 10 万，12 个月分期，还了 6 个月后违约：
- 已还本金 ≈ 5 万
- EAD ≈ 5 万（剩余未还本金）

**循环信贷（信用卡）**

信用卡的 EAD 更复杂，因为用户在违约前可能会把额度用满：

```
EAD = 当前已用额度 + 预期未来追加用款额
    = 当前已用额度 + 未用额度 × CCF
```

CCF（Credit Conversion Factor，信用转换系数）= 用户在违约前会把多少未用额度转化为实际负债。

**为什么考虑未使用额度？**

研究发现：即将违约的用户，在违约前往往会尽量"榨干"剩余额度（最后的资金来源），所以 EAD 通常高于当前已用余额。

## 五、三者结合：预期损失（EL）的计算

### 基础公式

```
EL = PD × LGD × EAD
```

**举例：**

某用户情况：
- PD = 5%（信用评分 680 分对应的违约概率）
- LGD = 75%（无担保消费贷，预期回收率 25%）
- EAD = 30,000 元（当前未偿还余额）

```
EL = 5% × 75% × 30,000 = 1,125 元
```

这意味着：这笔贷款，平均来说，预期损失 1,125 元。

### EL 在定价中的应用

利率定价必须覆盖预期损失：

```
最低利率 = 资金成本 + 运营成本 + EL/贷款金额 + 目标利润

例：
资金成本 = 4%（融资成本）
运营成本 = 2%（人工、系统等）
EL 成本  = 1125/30000 = 3.75%
目标利润 = 2%

最低利率 ≥ 4% + 2% + 3.75% + 2% = 11.75%
```

如果这笔贷款的利率低于 11.75%，从长期来看这个风险等级的客户群体会亏钱。

### 风险分层定价

不同风险等级的客户，EL 不同，定价也不同：

| 风险等级 | PD | LGD | EL（10万贷款） | 建议利率 |
|----------|----|----|----------------|----------|
| 低风险 | 2% | 75% | 1,500 元 | 10%–14% |
| 中风险 | 8% | 75% | 6,000 元 | 16%–20% |
| 高风险 | 20% | 75% | 15,000 元 | 24%–30% |

这就是"风险定价"的核心逻辑：高风险客户不是不能做，而是要收更高利率。

## 六、预期损失 vs 意外损失

### 预期损失（EL）

EL 是"正常情况下"的平均损失，是可以预测和定价的。

**应对方式**：纳入产品定价，通过收取足够的利息来覆盖。

### 意外损失（UL）

UL 是损失超出预期的部分——经济危机、自然灾害、行业崩溃等极端情况导致的损失。

**特点**：无法通过定价完全预测，需要储备资本金来吸收。

```
意外损失 = 最坏情景损失（如经济危机时）- 预期损失（正常情况）
```

**应对方式**：《巴塞尔协议》要求银行根据资产风险水平计提相应的**最低资本要求**，即用资本金作为缓冲，确保在极端情况下不会破产。

### 类比理解

- **预期损失** = 汽车的正常磨损（轮胎换新、机油更换），可以预算、纳入养车成本
- **意外损失** = 车祸（无法预测频率，但需要保险覆盖），类比资本金

## 七、PD、LGD、EAD 在实际工作中的体现

**在建模岗**：
- 建评分卡本质上是在估计每个用户的 PD
- LGD 估算需要单独建模或使用行业经验值

**在策略岗**：
- 根据 PD 设计准入阈值（PD > X% 的客户拒绝）
- 根据 EAD 设计额度上限（控制最大风险敞口）

**在数据分析岗**：
- 用历史数据验证 PD 预测的准确性（模型校准）
- 追踪 LGD 的实际值与预测值的差异

**在风险管理岗**：
- 汇总全部贷款的 EL，形成整体风险报告
- 测算极端情景下的 UL，评估资本充足率

---

## 小结

| 参数 | 核心问题 | 关键影响因素 | 应对工具 |
|------|----------|-------------|----------|
| PD | 他会不会违约？ | 征信历史、负债比、行为特征 | 评分卡（A/B卡） |
| LGD | 违约了损失多少？ | 抵押物、催收效率 | 催收体系、担保结构 |
| EAD | 违约时欠多少？ | 产品类型、额度利用率 | 额度管理 |
| EL | 整体预期损失？ | 三者的乘积 | 定价、拨备 |

理解这三个参数，是真正从业务视角理解信贷风控的起点。
