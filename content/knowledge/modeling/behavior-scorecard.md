## 一、为什么需要 B 卡？

A 卡只在申请时打一次分，但现实中，用户的风险会随时间变化：

- 用户在申请时如实填报了信息，A卡评分良好，但放款后失业了
- 用户申请时财务健康，但后来开始多头借贷，资金链出现问题
- 用户的消费行为发生异常，信用卡利用率突然飙升

**A卡只能告诉你"这个人申请时的风险"，而 B 卡告诉你"这个人现在的风险"。**

行为评分卡（Behavior Scorecard，B卡）持续使用用户放款后的行为数据，定期更新评分，动态反映风险变化。

---

## 二、A卡 vs B卡：核心区别

| 维度 | A 卡（申请评分卡） | B 卡（行为评分卡） |
|------|------|------|
| 使用时机 | 申请时，一次性使用 | 放款后，持续/定期更新 |
| 数据来源 | 申请表 + 征信查询 | 账户行为 + 还款记录 + 外部更新数据 |
| 预测目标 | 申请人是否会逾期 | 当前持卡人未来是否会逾期 |
| 更新频率 | 静态，不变 | 动态，通常月度更新 |
| 主要用途 | 准入决策 | 额度调整、预警触发、催收预判 |

**关键区别**：B 卡使用的特征是申请后才能产生的行为数据，这些数据对于申请评估来说不存在，但对于在贷用户却是最有价值的风险信号。

## 三、B 卡的特征体系

B 卡的特征可以分为三大类：

### 1. 账户内部行为特征（最重要）

这是 B 卡相比 A 卡最核心的优势——用户在你这里的真实行为。

**还款行为特征**

| 特征 | 说明 |
|------|------|
| 历史还款记录 | 是否按时还款，逾期几次 |
| 连续还款月数 | 连续不逾期的月数越多，越稳定 |
| 最近一次还款时间 | 距上次还款的时间间隔 |
| 还款金额占应还比例 | 是否全额还清，还是最低还款 |
| 最低还款比例 | 信用卡场景，习惯最低还款 = 资金紧张信号 |

**账户使用行为特征**

| 特征 | 说明 |
|------|------|
| 账户利用率（信用卡） | 已用额度/总额度，> 80% 是预警信号 |
| 近 N 期消费金额变化 | 消费突然萎缩可能意味着收入下降 |
| 取现比例（信用卡） | 取现比例高 = 流动性需求急迫 |
| 账户活跃度 | 是否在持续正常使用 |

**提前还款行为**

| 特征 | 说明 |
|------|------|
| 是否提前还款 | 提前还清通常是低风险信号 |
| 提前还款金额比例 | 大额提前偿还 vs 小额零散偿还含义不同 |

### 2. 外部行为特征（定期更新）

B 卡建模时，可以定期拉取用户的外部数据更新特征：

**征信更新数据**

| 特征 | 说明 |
|------|------|
| 近期在其他机构逾期情况 | 外部逾期 = 资金链开始断裂的早期信号 |
| 其他机构账户数变化 | 新开账户 = 开始多头借贷 |
| 近期征信查询次数 | 短期内多次查询 = 急迫的资金需求 |
| 当前总负债额变化 | 总负债增加 = 偿债压力上升 |

**外部数据源更新**

- 社保/公积金缴纳状态（是否断缴 = 失业信号）
- 电话号码活跃度（号码停机 = 联系风险）
- 多头借贷平台数据（同时申请多家 = 极高风险）

### 3. 交易与消费行为特征

对于有消费场景的产品（如信用卡）：

| 特征 | 说明 |
|------|------|
| 消费品类变化 | 医疗类消费增加可能是突发事件 |
| 消费地理位置变化 | 突然出现异地消费 |
| 交易频率 | 交易突然减少 |

## 四、B 卡的建模流程

### 样本选择

B 卡与 A 卡在样本选择上有重要差异：

**观察点**：通常选在放款后某个固定时间点（如放款后 6 个月），此时已积累了足够的行为数据。

**表现期**：从观察点开始，往后看 6–12 个月的还款表现。

**关键问题：账龄（MOB）的影响**

MOB（Months on Books）是用户放款后的月数。不同 MOB 的用户，行为特征和风险特征差异很大：

- MOB 1–3：刚放款，行为数据很少，还款压力未完全体现
- MOB 4–12：行为数据充足，是建模的黄金期
- MOB 13+：部分用户已接近还清，风险趋于下降

> 如果不区分 MOB 直接建模，会混入大量行为特征极少的早期账户，降低模型效果。解决方案：**分 MOB 段建模**，或将 MOB 作为分群变量。

### 好坏定义

B 卡的好坏定义与 A 卡类似，但通常更严格：

- 常见定义：从观察点开始，未来 6–12 个月内出现 M2+（逾期 60 天以上）视为坏
- 对于信用卡：过去 12 个月内出现 2 次 M1+ 视为坏

### 建模算法

B 卡同样以逻辑回归为主，理由与 A 卡相同（可解释性、稳定性）。

特征工程步骤：分箱 → WOE 编码 → IV 筛选 → 逻辑回归 → 评分转换（详见申请评分卡章节）。

## 五、B 卡的动态更新机制

B 卡不是建一次就不变的，需要定期更新评分：

### 评分更新频率

| 场景 | 更新频率 |
|------|----------|
| 信用卡 | 月度更新（每次出账单后） |
| 消费贷 | 月度或季度更新 |
| 大额长期贷款 | 季度更新 |

### 评分随 MOB 的演变

同一个用户，随着还款月数增加，评分会发生变化：

```
MOB 1    MOB 6    MOB 12   MOB 18
  ↓        ↓        ↓        ↓
行为稀少  行为丰富  风险清晰  趋于稳定
评分较随机 评分准确  评分稳定  预测价值下降
```

B 卡价值最高的阶段是 MOB 4–12，这个阶段行为数据充足，距离还清还有足够时间采取措施。

## 六、B 卡的应用场景

### 1. 额度管理

基于 B 卡评分，对存量用户进行动态额度调整：

| B 卡评分变化 | 策略 |
|------|------|
| 持续高分 | 主动增额（提升用户黏性） |
| 评分无显著变化 | 维持现有额度 |
| 评分下降 20–30 分 | 预警，不增额，观察 |
| 评分大幅下降 | 主动降额或冻结 |

### 2. 贷中预警

B 卡评分触发预警系统的核心指标之一：

```
B 卡评分 < 预警阈值
    ↓
推送到预警队列
    ↓
按降幅分级处理：
  轻度下降 → 系统监控，增加数据刷新频率
  中度下降 → 发送关怀触达，了解用户情况
  重度下降 → 人工跟进，控制风险敞口
```

### 3. 催收优先级排序

当用户逾期时，B 卡历史评分曲线提供了重要上下文：

- **评分一直稳定 → 突然逾期**：可能是偶发事件（忘记还款），出催概率高
- **评分持续下滑 → 最终逾期**：风险持续积累，催收难度大

这正是 C 卡（催收评分卡）建模时会纳入 B 卡分作为特征的原因。

## 七、B 卡的常见误区

**误区1：B 卡准确率一定比 A 卡高**

不一定。B 卡特征更丰富，但预测的是"当前状态下未来的风险"，而用户情况在未来仍可能发生突变（如突然失业）。B 卡的价值更多体现在对存量客户的实时预警，而非预测精度的绝对提升。

**误区2：B 卡可以完全取代 A 卡**

不能。A 卡在申请时使用，B 卡需要历史行为数据，申请时尚不存在。两者覆盖不同阶段，互补而非替代。

**误区3：B 卡分只用于风控**

B 卡分也是营销工具。高 B 卡分的用户是增额、推荐新产品的首选目标；B 卡分稳定且高的用户，是最有价值的客户，值得投入资源维系关系。

---

## 小结

B 卡是 A 卡在时间维度上的延伸，将风控从"申请时一次性判断"升级为"全生命周期持续监控"。

| 阶段 | 工具 | 核心问题 |
|------|------|----------|
| 申请时 | A 卡 | 要不要批？批多少？ |
| 放款后 | B 卡 | 现在风险有没有变？要不要调整策略？ |
| 逾期后 | C 卡 | 能不能催回来？怎么催最有效？ |

三张评分卡共同构成信贷风控的完整模型体系。
