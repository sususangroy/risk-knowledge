## 一、什么是申请评分卡？

当一个人向金融机构申请贷款时，机构需要在几秒钟内决定：**借还是不借？借多少？**

申请评分卡（Application Scorecard，简称 A卡）就是做这个决策的自动化工具。

它的输入是申请人在申请时提交的信息（个人基本信息、收入、征信记录等），输出是一个**信用分数**。分数越高，代表此人未来还款的可能性越大，被批准的概率也越高。

**A卡在风控体系中的位置：**

```
申请进来
  ↓
反欺诈模型（先排除欺诈申请）
  ↓
硬规则过滤（强制拒绝：如征信有当前逾期）
  ↓
A卡评分（连续打分，按分数决定通过/拒绝/人工审核）
  ↓
放款
```

---

## 二、为什么用评分卡而不是人工审核？

在信贷规模很小时，银行会让信贷员逐个审核申请。但当每天有几万甚至几百万申请时，人工审核不可能。

评分卡有几个核心优势：

1. **一致性**：同样的信息，永远给出同样的分数，不受人的情绪、疲劳影响
2. **可解释性**：能告诉申请人"因为收入不达标而被拒"，满足监管的公平贷款要求
3. **可优化**：模型效果可量化（KS、AUC），便于持续迭代
4. **速度**：毫秒级决策，支撑高并发场景

## 三、建模前的核心概念

### 好客户 vs 坏客户（好坏定义）

建模的第一步是**定义谁是"坏客户"**。这不是一个技术问题，而是一个业务决策。

常见定义：
- **宽松定义**：逾期 30 天以上（M1+）视为坏
- **严格定义**：逾期 90 天以上（M3+）视为坏
- **信用卡常用**：过去 24 个月内有过 2 次以上 30 天逾期

> 一旦好坏定义确定，整个建模过程就固定了。好坏定义的变化会导致模型需要重建。

### 观察点（Observation Point）与表现期（Performance Window）

这两个概念是理解建模逻辑的关键：

- **观察点**：我们用什么时刻的信息来预测？（通常是申请日）
- **表现期**：我们观察申请人在多长时间内的还款表现？（通常是 6–24 个月）

举例：2023 年 1 月申请的用户，用申请时的信息作为特征，看其 2023 年 1 月 ~ 2024 年 1 月（12 个月）的还款情况来打标签。

```
  申请日（观察点）             表现期结束
  ┌───────────────────────────────────┐
  │ 特征X ──→ 表现Y（好/坏）          │
  └───────────────────────────────────┘
         ←──── 表现期（如12个月） ─────→
```

### 样本成熟度问题

表现期需要足够长（通常 12–24 个月），这意味着：

- 用最近 3 个月申请的用户建模，他们还没走完表现期，坏样本未充分暴露 → **右删失（Right Censoring）**问题
- 解决方案：剔除表现期内已离开（提前还清）的样本；或使用存活分析方法

## 四、建模数据准备

### 训练样本窗口的选择

不能只用某一个月的申请用户，需要选择一段时间窗口（通常 6–24 个月）的申请样本，原因：

- 单月数据量可能不足
- 单月数据可能有季节性偏差（如春节前申请的用户群体特殊）
- 需要覆盖不同客群结构

**窗口选择的注意事项：**
1. 窗口内产品/策略变化不能太大（否则客群分布不一致）
2. 尾部样本表现期不足，需截断或标记
3. 时间越近的样本，越能反映当前客群，可加权处理

### 样本排除规则

以下样本通常需要在建模前排除：
- 表现期内数据缺失或系统异常的
- 欺诈案件（欺诈用逃离而非信用风险导致逾期，样本性质不同）
- 极小额贷款（业务性质特殊）

## 五、特征工程

### A卡常用特征

**基本信息类**

| 特征 | 说明 |
|------|------|
| 年龄 | 年轻人违约率较高，中年最稳定，老年存在收入下滑风险 |
| 性别 | 在某些产品中有区分度，但需注意公平贷款合规 |
| 婚姻状况 | 已婚通常更稳定（双收入、家庭责任感） |
| 学历 | 与收入稳定性相关 |
| 居住状况 | 自有房产 vs 租房，稳定性不同 |

**收入与负债类**

| 特征 | 说明 |
|------|------|
| 月收入（自填） | 最直接的还款能力指标，但存在高报风险 |
| 收入验证 | 社保缴费基数、公积金缴纳额、银行流水 |
| 月供/收入比（DTI） | Debt-to-Income Ratio，负债率越高风险越大 |
| 外部多头借贷 | 同时在多家机构有贷款，是重要风险信号 |

**征信类（最重要）**

| 特征 | 说明 |
|------|------|
| 历史逾期记录 | 过去逾期是预测未来逾期最强的特征 |
| 历史逾期次数/天数 | 轻微偶发 vs 严重惯性逾期差别很大 |
| 信用账户数量 | 账户过多可能是多头借贷信号 |
| 最近一次逾期时间 | 越近越危险，很久以前可能已经恢复 |
| 账户利用率 | 信用卡已用额度/总额度，越高压力越大 |
| 征信查询次数 | 短期内大量查询 = 急需用钱，风险提示 |

**贷款申请行为类**

| 特征 | 说明 |
|------|------|
| 贷款申请次数（近3/6/12个月） | 申请频次突增是高风险信号 |
| 申请额度 vs 实际需求 | 申请额度远超合理需求，可能有欺诈或过度借贷 |
| 申请时间（工作日/深夜） | 深夜申请的部分产品逾期率更高 |

### WOE 编码（Weight of Evidence，证据权重）

这是评分卡建模最核心的特征处理步骤，将原始变量转换为 WOE 值。

**为什么要做 WOE 编码？**

1. 处理非线性关系（如年龄与逾期率可能是 U 型曲线）
2. 统一不同量纲的变量（收入单位是元，年龄单位是岁）
3. 自动处理缺失值（缺失值单独一个箱）
4. 提高模型可解释性

**WOE 的计算方式：**

首先对变量进行分箱（Binning），每个箱计算：

```
WOE_i = ln( 好样本率_i / 坏样本率_i )
      = ln( (好样本数_i / 总好样本数) / (坏样本数_i / 总坏样本数) )
```

直觉理解：
- WOE > 0：该箱内好客户比例高于整体，是正向信号
- WOE < 0：该箱内坏客户比例高于整体，是负向信号
- WOE = 0：无区分度

**举例：**

假设"近3个月逾期次数"的分箱结果：

| 分箱 | 好样本率 | 坏样本率 | WOE | 含义 |
|------|----------|----------|-----|------|
| 0次 | 60% | 30% | +0.69 | 从未逾期，强正信号 |
| 1次 | 25% | 35% | -0.34 | 轻微负信号 |
| 2次及以上 | 15% | 35% | -0.85 | 强负信号 |

### IV 值（Information Value，信息价值）

IV 值衡量一个变量对好坏的总体区分能力：

```
IV = Σ (好样本率_i - 坏样本率_i) × WOE_i
```

**IV 值的参考标准：**

| IV 值范围 | 解读 |
|-----------|------|
| < 0.02 | 无预测能力，通常剔除 |
| 0.02 – 0.1 | 弱预测能力 |
| 0.1 – 0.3 | 中等预测能力 |
| > 0.3 | 强预测能力 |

## 六、模型训练

### 为什么使用逻辑回归（Logistic Regression）？

评分卡虽然可以用 XGBoost、随机森林等复杂算法，但实际中绝大多数金融机构的 A 卡仍然用**逻辑回归**，原因：

1. **可解释性**：逻辑回归的系数有明确的业务含义，可以逐条解释每个变量对分数的贡献
2. **监管要求**：银行等持牌机构需要向监管解释"为什么拒绝某申请人"，黑盒模型无法满足
3. **稳定性**：逻辑回归对数据分布变化不敏感，部署后更稳定
4. **已足够好**：在做好 WOE 编码和特征筛选后，逻辑回归的效果与复杂模型差距有限

### 特征筛选原则

进入最终模型的变量需满足：

1. **IV 值 > 0.02**（有信息量）
2. **WOE 单调性**（分箱后 WOE 随特征值单调变化，保证逻辑一致）
3. **相关性检查**（变量之间相关系数 < 0.7，避免多重共线性）
4. **业务合理性**（模型输出需能被业务解释，不能因为 IV 高就盲目纳入）

## 七、模型评估

### KS 值（Kolmogorov-Smirnov）

KS 值衡量模型对好坏客户的区分能力：

```
KS = max(好客户累积分布 - 坏客户累积分布)
```

**直觉解释**：按评分从低到高排列所有人，如果模型有区分度，坏客户会集中在低分段，好客户集中在高分段。KS 就是在哪个分数点，两者的累积分布差最大。

| KS 值 | 评价 |
|-------|------|
| < 0.2 | 差，模型几乎无用 |
| 0.2 – 0.3 | 一般 |
| 0.3 – 0.5 | 良好，大多数生产模型在此区间 |
| > 0.5 | 优秀，需检查是否存在过拟合或数据泄露 |

### AUC（Area Under the ROC Curve）

AUC 反映随机抽取一个好客户和一个坏客户，好客户评分更高的概率：

- AUC = 0.5：完全随机，无区分能力
- AUC = 1.0：完美区分
- 实际 A 卡目标：AUC > 0.7，优秀模型可达 0.75–0.85

### 分数分布检查

除了 KS/AUC，还需检查：
- 评分分布是否合理（不能过度集中在某个分段）
- 好坏客户评分分布是否有明显分离
- 各关键子群体（如新户 vs 老户）的评分是否合理

## 八、评分转换：从概率到分数

逻辑回归直接输出的是违约概率（0–1 之间），但业务上通常需要一个整数分数，原因是直觉更友好。

**标准评分卡转换公式：**

```
Score = A - B × ln(odds)
```

其中 odds = 好样本数 / 坏样本数，A、B 为缩放参数，通过以下设定求解：
- 设定基准点：某个 odds 对应的分数（如 odds=50 时分数=600）
- 设定 PDO（Points to Double Odds）：odds 翻倍时分数增加量（如 PDO=20）

这样设计后，**每增加 20 分，违约概率降低一半**，业务方一看到分数就能感知风险变化幅度。

## 九、A卡部署与策略应用

### 分数切割策略

A 卡上线后，需要根据分数设置准入策略：

| 分数段 | 策略 | 说明 |
|--------|------|------|
| 高分（如 ≥ 680） | 自动通过 | 风险低，快速审批 |
| 中分（如 620–680） | 人工复审 | 有一定风险，需补充材料核查 |
| 低分（如 < 620） | 自动拒绝 | 风险超过可接受阈值 |

切割点的选择取决于机构的风险偏好（通过率目标 vs 坏账容忍度）。

### A 卡与规则的配合

A卡不是孤立使用的，通常与规则引擎配合：

```
进件
 ↓
硬规则（强制拒绝）：如"当前有逾期" → 拒绝
 ↓
A卡打分
 ↓
按分数分层：高分段可降低人工复核率；低分段直接拒绝
 ↓
其他策略叠加（如申请金额限制、需上传收入证明）
```

## 十、A卡的局限性

A卡很强大，但也有局限：

1. **静态性**：A卡只用申请时的信息，无法反映放款后的风险变化 → 这是 B卡（行为评分卡）存在的原因
2. **依赖历史数据**：新客群、新产品上线初期，训练数据不足，A卡效果受限
3. **信息自报偏差**：申请人填写的收入、职业等信息可能不真实，需要外部数据核验
4. **不含欺诈信号**：欺诈用户可能在 A 卡上得高分（他们会伪造信息），需要独立的反欺诈模型

## 小结

申请评分卡（A 卡）的建模流程可以概括为：

```
业务定义好坏 → 选择观察点/表现期 → 准备训练样本
    ↓
特征收集（征信/收入/行为）→ 分箱 → WOE 编码 → IV 筛选
    ↓
逻辑回归建模 → KS/AUC 评估 → 评分转换
    ↓
策略设计（分数切割/规则配合）→ 上线部署 → 持续监控
```

掌握 A 卡的核心，就掌握了贷前风控的骨架。后续的 B 卡、C 卡都是在相同的框架下，针对不同阶段的问题进行的专项优化。
