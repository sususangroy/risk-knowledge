## 一、评分卡为什么是信贷风控的标准方法？

评分卡本质上是一个简化的线性模型：对每个特征的不同取值赋予一定分数，所有特征分数加总就是最终得分。

**评分卡能持续流行的原因：**

1. **可解释性**：可以告诉申请人"因为您近3个月有逾期记录，扣了 30 分"
2. **监管合规**：银行等持牌机构必须能够解释拒绝原因，黑盒模型无法满足
3. **稳定性**：逻辑回归对分布变化不敏感，部署后不会突然失效
4. **易于更新**：新增一个变量或调整分箱，不需要重新训练整个模型
5. **效果足够好**：经过好的特征工程后，评分卡的效果通常与复杂模型相差不大

## 二、开发前的准备

### 明确建模目标

在开始任何工作之前，必须回答三个问题：

**问题1：用于什么场景？**
- A 卡：贷前准入决策
- B 卡：贷中风险监控和额度调整
- C 卡：贷后催收优先级

**问题2：好坏怎么定义？**

这是最重要的业务决策，不同定义会产生完全不同的模型：

| 定义严格程度 | 常见定义 | 适用场景 |
|------------|----------|----------|
| 宽松 | 逾期 30 天（M1+） | 短期贷款、信用卡 |
| 中等 | 逾期 60 天（M2+） | 中期消费贷 |
| 严格 | 逾期 90 天（M3+） | 长期贷款、银行标准 |

> 定义越宽松，坏样本越多，模型训练更容易但可能过于敏感；定义越严格，坏样本越少，模型更难训练但关注真正严重的违约。

**问题3：表现期多长？**

表现期 = 观察用户行为的时间窗口：
- 太短（< 6 个月）：很多违约还没来得及发生，坏样本不充分
- 太长（> 24 个月）：训练数据过于历史化，不能反映当前客群
- **通常建议：12–18 个月**

## 三、样本准备

### 3.1 观察点与表现期的设计

```
申请时间（观察点）      表现期结束
    |                      |
    ●──────────────────────●
    ↑                      ↑
  提取特征X              打标签Y（好/坏）
```

**关键原则：特征必须来自观察点之前，标签来自观察点之后。**

常见错误：把放款后才能知道的信息（如首期实际还款金额）作为特征，这叫**数据泄露（Data Leakage）**，会让模型在训练集上看起来很好，实际部署后完全失效。

### 3.2 训练样本窗口的选择

不能只用某一个月的数据，原因：
- 单月数据量通常不足
- 单月可能有季节性偏差（如春节前后客群特殊）
- 需要覆盖不同时期的客群结构

**通常选择 6–18 个月的申请样本**，确保：
- 窗口内产品/策略没有重大变化
- 最老的样本表现期已经完整（已经看到了足够的违约行为）
- 最新的样本表现期足够长（尾部截断需要处理）

### 3.3 好坏样本比例

信贷数据通常是**不平衡的**（坏样本 << 好样本）：

| 产品类型 | 典型坏账率 | 好坏比例 |
|----------|-----------|----------|
| 优质消费贷 | 2%–5% | 约 1:20 到 1:50 |
| 中等风险产品 | 8%–15% | 约 1:7 到 1:12 |
| 高风险小额贷 | 15%–30% | 约 1:3 到 1:7 |

当坏样本太少时（< 3%），可以考虑：
- 对坏样本过采样（Oversampling）
- 对好样本欠采样（Undersampling）
- 使用类别权重（在逻辑回归中设置 class_weight）

### 3.4 需要排除的样本

- **欺诈样本**：欺诈导致的逾期与信用风险性质不同，混入会污染模型
- **系统异常样本**：因系统故障导致的异常还款记录
- **政策特殊处理样本**：如新冠期间的延期还款政策，不能代表正常违约行为

## 四、特征工程

### 4.1 分箱（Binning）

在 WOE 编码之前，需要对连续变量进行分箱，将其离散化为若干区间。

**为什么分箱？**
- 消除极端值的影响
- 捕捉非线性关系（年龄与风险可能是 U 型关系）
- 将缺失值作为单独一个箱处理

**分箱方法：**

| 方法 | 原理 | 适用场景 |
|------|------|----------|
| 等频分箱 | 每箱样本数量相等 | 初步探索，均匀覆盖 |
| 等距分箱 | 每箱区间宽度相等 | 直觉明确的变量（如年龄） |
| 最优分箱 | 按 WOE 单调性最优化分箱 | 建模阶段，使用卡方或树基方法 |

**好分箱的判断标准：**
- 每个箱子样本量足够（通常 ≥ 5% 的总样本）
- WOE 值单调（随变量值增大，WOE 单调变化）
- 分箱数量适中（5–10 箱，太多不稳定，太少信息损失）
- 缺失值单独成一箱

**示例：月收入的分箱**

| 分箱 | 月收入范围 | 样本数 | 坏账率 | WOE |
|------|-----------|--------|--------|-----|
| 箱1 | < 3000 | 1200 | 18% | -0.92 |
| 箱2 | 3000–6000 | 3500 | 10% | -0.12 |
| 箱3 | 6000–10000 | 4000 | 6% | +0.45 |
| 箱4 | 10000–20000 | 2800 | 3.5% | +0.88 |
| 箱5 | > 20000 | 500 | 2% | +1.15 |
| 缺失 | 无收入信息 | 800 | 20% | -1.04 |

WOE 值随收入单调递增 → 这是一个好的分箱。

### 4.2 WOE 编码

**计算方式：**

```
好样本率_i = 该箱好样本数 / 总好样本数
坏样本率_i = 该箱坏样本数 / 总坏样本数

WOE_i = ln(好样本率_i / 坏样本率_i)
```

**直觉理解：**
- WOE > 0：这个箱内好客户的占比高于整体平均 → 正面信号
- WOE < 0：这个箱内坏客户的占比高于整体平均 → 负面信号
- WOE 绝对值越大：区分度越强

**WOE 编码的好处：**
- 统一量纲：所有特征都转换为同一量级，可以直接比较系数大小
- 处理缺失值：缺失值有对应的 WOE，不需要填充
- 线性化：逻辑回归假设特征与 log(odds) 线性，WOE 编码有助于满足这一假设

### 4.3 IV 值（Information Value）

IV 值衡量变量的整体预测能力：

```
IV = Σ (好样本率_i - 坏样本率_i) × WOE_i
```

**判断标准：**

| IV 值 | 预测能力 | 处理建议 |
|-------|----------|----------|
| < 0.02 | 几乎无效 | 直接剔除 |
| 0.02–0.1 | 弱 | 结合业务判断是否保留 |
| 0.1–0.3 | 中等 | 通常保留 |
| 0.3–0.5 | 强 | 保留，重点变量 |
| > 0.5 | 极强 | 谨慎！可能是数据泄露 |

> **IV > 0.5 要警惕**：极高的 IV 可能意味着这个特征"偷看"了标签，比如用"逾期状态"来预测"是否违约"，这两个本质上是同一件事。

## 五、逻辑回归建模

### 5.1 特征筛选

在所有 WOE 编码后的特征中，最终选入模型的变量需满足：

**定量标准：**
- IV > 0.02（有信息量）
- 特征之间 Pearson 相关系数 < 0.7（避免多重共线性）
- 逐步回归后系数方向与业务含义一致（月收入越高应该是正向，不能出现负系数）

**定性标准（业务审查）：**
- 特征在业务上可以解释
- 特征在部署时能实时获取
- 特征不涉及受保护属性（性别、种族等歧视性特征）

### 5.2 模型训练

```python
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(
    X_woe, y, test_size=0.3, random_state=42, stratify=y
)

model = LogisticRegression(
    C=1.0,           # 正则化系数，越小正则化越强
    solver='lbfgs',
    max_iter=1000
)
model.fit(X_train, y_train)
```

**训练-测试集划分注意事项：**
- 使用 `stratify=y` 确保训练集和测试集中好坏比例相同
- 对于时间序列数据，应按时间划分（早期训练，晚期测试），而不是随机划分

### 5.3 过拟合检查

比较训练集和测试集的 KS 差异：
- 差异 < 5%：正常
- 差异 5%–10%：轻微过拟合，可接受
- 差异 > 10%：过拟合较严重，考虑增加正则化或减少特征

## 六、模型评估

### 6.1 KS 值

**计算方式：**

按评分从低到高排列所有样本，逐步计算：
- 好客户的累积分布曲线
- 坏客户的累积分布曲线

KS = 两条曲线之间的最大差值

**直觉：** 如果模型把坏客户都排在低分段，好客户都在高分段，两条曲线会快速分开，KS 高。如果模型无法区分，两条曲线几乎重叠，KS ≈ 0。

| KS 值 | 模型质量 |
|-------|----------|
| < 0.2 | 差，基本无用 |
| 0.2–0.3 | 一般 |
| 0.3–0.5 | 良好（大多数生产模型） |
| 0.5–0.7 | 优秀 |
| > 0.7 | 极好，需核查是否有数据泄露 |

### 6.2 AUC（ROC 曲线下面积）

AUC 的直觉解释：**随机抽取一个好客户和一个坏客户，好客户评分更高的概率。**

- AUC = 0.5：完全随机（掷硬币）
- AUC = 1.0：完美区分
- 实际 A 卡目标：AUC ≥ 0.7

### 6.3 评分分布检查

除了 KS/AUC，还需要检查：

**评分分布的形状**：是否大致正态分布？如果极度集中在某个分数段，策略切割会很困难。

**各分数段好坏比（Odds）是否单调**：高分段 Odds 应该显著高于低分段，如果某个分数段出现反转，说明模型有局部问题。

**分群一致性检查**：分别在男女、新老用户、不同渠道上检查 KS，模型在各子群上应该都有基本的区分能力。

## 七、评分转换

将逻辑回归输出的违约概率转换为业务友好的整数分数。

### 转换公式

```
Score = A - B × ln(Odds)

其中：Odds = P(好) / P(坏) = (1-p) / p
```

**参数设置（以常见设定为例）：**
- 基准点：Odds = 50 时，Score = 600
- PDO = 20（Points to Double Odds，odds 翻倍时分数增加 20 分）

**反推参数：**
```
B = PDO / ln(2) ≈ 20 / 0.693 ≈ 28.85
A = 600 + 28.85 × ln(50) ≈ 600 + 28.85 × 3.912 ≈ 712.9
```

**效果：**
- 每增加 20 分，违约概率大约降低一半
- 600 分：odds = 50（好坏比 50:1）
- 620 分：odds = 100（好坏比 100:1）
- 580 分：odds = 25（好坏比 25:1）

### 评分卡展示格式

最终的评分卡通常以"得分表"的形式展示给业务方：

| 变量 | 分箱 | 得分 |
|------|------|------|
| 月收入 | < 3000 | -25 |
|  | 3000–6000 | -5 |
|  | 6000–10000 | +12 |
|  | > 10000 | +23 |
| 近3个月逾期次数 | 0 次 | +18 |
|  | 1 次 | -8 |
|  | 2次及以上 | -28 |
| ... | ... | ... |
| **基础分** | | **600** |

用户最终得分 = 600 + 各变量得分之和

## 八、模型上线与监控

### 上线前检查清单

- [ ] 测试集 KS 与训练集 KS 差距 < 10%（无过拟合）
- [ ] 评分分布合理，无极端集中
- [ ] 所有特征在生产环境中可以实时获取
- [ ] 缺失值处理逻辑已在线上实现
- [ ] 回测分析完成，业务方认可通过率和预期坏账变化

### 上线后监控

| 监控指标 | 频率 | 预警阈值 |
|----------|------|----------|
| PSI（评分分布稳定性） | 月度 | > 0.2 触发审查 |
| KS 值变化 | 季度 | 下降 > 15% 触发审查 |
| 各分数段实际坏账率 | 季度 | 与预期偏差 > 30% |

详细监控方法请参见《模型监控体系（PSI/KS）》。

## 小结

评分卡开发的完整流程：

```
业务定义
(好坏/观察点/表现期)
    ↓
样本准备
(训练窗口/排除规则/好坏比处理)
    ↓
特征工程
(分箱 → WOE编码 → IV筛选)
    ↓
模型训练
(逻辑回归 → 特征筛选 → 过拟合检查)
    ↓
模型评估
(KS/AUC/评分分布检查)
    ↓
评分转换
(概率 → 分数)
    ↓
上线部署
(回测 → AB测试 → 监控)
```

每一步都有对应的业务逻辑和技术细节。掌握这个完整流程，就掌握了信贷建模工作的核心框架。
