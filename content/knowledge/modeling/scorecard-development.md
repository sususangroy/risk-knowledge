## 评分卡的地位

评分卡（Scorecard）是信贷风控建模中最经典的方法，具有**可解释性强、易于业务落地**的优势，至今仍是主流金融机构的核心模型之一。

## 开发流程

### 1. 样本定义

**观察窗口（Observation Window）：** 用于提取特征的时间段

**表现窗口（Performance Window）：** 用于定义好坏标签的时间段

```
|——观察期——|——表现期——|
  特征提取      标签定义
```

**好坏定义（Bad Definition）：** 通常以 M3（逾期 90 天以上）作为"坏"的定义。

### 2. 数据处理与变量筛选

**缺失值处理：**
- 少量缺失：用均值/众数填充
- 大量缺失（>50%）：考虑单独作为一组处理，或删除变量

**IV 值（Information Value）筛选：**

| IV 值范围 | 变量预测能力 |
|----------|------------|
| < 0.02 | 几乎无预测能力 |
| 0.02 ~ 0.1 | 弱预测能力 |
| 0.1 ~ 0.3 | 中等预测能力 |
| > 0.3 | 强预测能力（注意是否过拟合）|

通常保留 IV > 0.05 的变量进入后续流程。

### 3. WOE 编码

WOE（Weight of Evidence）将连续变量离散化后编码：

```
WOE_i = ln(坏客户比例_i / 好客户比例_i)
```

WOE 编码的优势：
- 处理非线性关系
- 处理缺失值（单独作为一个分箱）
- 使模型具有业务可解释性

**分箱原则：**
- 每个分箱样本量不少于 5%
- WOE 值单调（便于解释）
- 分箱数量通常 5~10 个

### 4. 逻辑回归建模

WOE 编码后，对各变量使用逻辑回归：

```python
from sklearn.linear_model import LogisticRegression
model = LogisticRegression(C=1.0, solver='lbfgs')
model.fit(X_woe_train, y_train)
```

### 5. 模型评估

**核心指标：**

- **AUC（Area Under Curve）：** 模型整体区分能力，越接近 1 越好
  - AUC > 0.75：可用
  - AUC > 0.8：良好
  
- **KS（Kolmogorov-Smirnov）：** 好坏客户分布的最大差异
  - KS > 0.2：可用
  - KS > 0.4：优秀

- **PSI（Population Stability Index）：** 监控模型稳定性
  - PSI < 0.1：稳定
  - PSI > 0.25：需重新训练

### 6. 分数转换

将逻辑回归输出的概率映射到评分（通常 300~850 分）：

```
Score = A - B × ln(Odds)
```

其中 A、B 为缩放参数，Odds = p/(1-p)，p 为坏客户概率。

## 模型上线后监控

- 每月监控 PSI（稳定性）
- 每季度监控 AUC/KS（区分能力）
- 出现 PSI > 0.2 时触发模型重训
