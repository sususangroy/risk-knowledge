## 一、为什么风控偏爱逻辑回归

在机器学习算法百花齐放的今天，逻辑回归（Logistic Regression）仍然是信贷风控最常用的基础模型。原因有三：

1. **结果可解释**：每个特征的系数直接说明该变量对违约概率的正负影响，监管机构和业务团队都能看懂
2. **输出为概率**：天然输出 0–1 之间的违约概率，可直接转换为风险评分
3. **稳定性好**：相比复杂模型不容易过拟合，在特征工程到位的情况下线上效果稳健

> 传统评分卡（Application Scorecard、Behavior Scorecard）的数学本质就是经过 WOE 变换后的逻辑回归。

## 二、数学原理（不怕数学版）

### 问题定义

在信贷场景中，我们要预测一个客户是否会逾期（坏客户 = 1，好客户 = 0）。这是一个**二分类**问题。

普通线性回归预测的是一个连续值，可能输出 -0.3 或 1.5，无法当作概率使用。逻辑回归的做法是把线性预测值"压缩"到 (0, 1) 区间。

### Sigmoid 函数

逻辑回归使用 **Sigmoid 函数** 做这个压缩：

```
P(违约) = 1 / (1 + e^(-z))

其中：z = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ
```

- `x₁, x₂, ...` 是特征（如年龄、收入、历史逾期次数）
- `w₁, w₂, ...` 是对应的系数（权重）
- z 越大，违约概率越高；z 越小，违约概率越低

### 直觉理解

想象一条 S 形曲线：
- z 极大时（多个高风险特征叠加）→ 违约概率趋近于 1
- z 极小时（多个低风险特征叠加）→ 违约概率趋近于 0
- z = 0 时 → 违约概率恰好 50%

## 三、WOE 编码：为什么风控偏爱它

### 什么是 WOE

WOE（Weight of Evidence，证据权重）是一种专为逻辑回归设计的特征编码方式，把原始特征转换为能反映"好坏区分能力"的数值。

```
WOE = ln(坏客户占比 / 好客户占比)

其中：
- 坏客户占比 = 该分箱内坏客户数 / 全体坏客户数
- 好客户占比 = 该分箱内好客户数 / 全体好客户数
```

**直觉**：如果某个年龄段内坏客户比例远高于整体，WOE 为正值（风险高）；反之为负值（风险低）。

### WOE + 逻辑回归的优势

| 优势 | 说明 |
|------|------|
| 处理非线性关系 | 通过分箱 + WOE，逻辑回归也能捕捉非线性模式 |
| 处理缺失值 | 缺失值单独作为一个分箱，有其 WOE 值 |
| 提升可解释性 | 每个特征的贡献是线性叠加的，可以直接解读 |
| 评分卡转换 | WOE 变换后，系数 × WOE 值 + 截距 可直接生成评分 |

### IV 值：特征筛选的标准

在做逻辑回归之前，通常用 **IV（Information Value，信息值）** 筛选有区分能力的特征：

```
IV = Σ (坏客户占比 - 好客户占比) × WOE
```

| IV 值范围 | 区分能力 |
|-----------|---------|
| < 0.02 | 几乎没有预测能力，丢弃 |
| 0.02 – 0.1 | 弱预测能力 |
| 0.1 – 0.3 | 中等预测能力，可使用 |
| 0.3 – 0.5 | 强预测能力 |
| > 0.5 | 非常强，需检查是否有数据泄露 |

## 四、模型训练要点

### 正则化：防止过拟合

#### 为什么需要正则化

逻辑回归的训练目标是最小化**对数损失函数（Log Loss）**：

```
Loss = -1/N × Σ [ y_i × log(p_i) + (1 - y_i) × log(1 - p_i) ]

  N   ：样本数量
  y_i ：第 i 个样本的真实标签（1=坏客户，0=好客户）
  p_i ：模型预测的违约概率
```

当特征数量多、样本量相对不足时，模型会倾向于给每个特征赋予较大系数以完美拟合训练集，导致**过拟合**：训练集表现好，但在新数据上泛化能力差。

正则化在损失函数中加入对系数大小的惩罚项，迫使模型保持系数尽量小，从而抑制过拟合。

---

#### L1 正则（Lasso）

在损失函数中加入所有系数**绝对值之和**作为惩罚：

```
J(w) = Loss + λ × Σ|w_j|

  λ（lambda）：正则化强度，越大惩罚越重
  w_j        ：第 j 个特征的系数
```

**核心特性：稀疏化——将不重要特征的系数精确压缩为 0**

L1 惩罚项在系数为 0 处不可导，梯度下降会将小系数"硬推"到 0，实现自动变量筛选。模型最终只保留若干关键特征，其余系数清零。

**适用场景**：
- 候选特征数量极多（数百至数千），需要自动稀疏化
- 希望模型本身承担部分变量筛选工作
- 可解释性要求：最终系数非零的变量即为入模变量

---

#### L2 正则（Ridge）

在损失函数中加入所有系数**平方和**作为惩罚：

```
J(w) = Loss + λ × Σw_j²

  λ：正则化强度
  w_j：第 j 个特征的系数
```

**核心特性：收缩——将所有系数均匀压缩，但不会压为 0**

L2 惩罚项处处可导，梯度下降会按比例缩小每个系数，而不会将任何系数精确清零。多个相关特征的系数会被均匀分摊，而非集中到其中一个。

**适用场景**：
- 特征之间存在多重共线性（如多个同类行为特征）
- 希望所有变量都保留，只压缩其影响力
- 风控评分卡建模中最常用，系数稳定性更好

**L1 与 L2 的对比**：

| 维度 | L1（Lasso） | L2（Ridge） |
|------|------------|------------|
| 惩罚项 | λ × Σ\|w_j\| | λ × Σw_j² |
| 系数效果 | 部分系数精确为 0（稀疏） | 所有系数均匀缩小（不为 0） |
| 变量筛选 | 自动筛选 | 不筛选，需手动剔除 |
| 共线性处理 | 从相关变量中随机保留一个 | 相关变量系数均匀分摊 |
| 风控常用度 | 中（辅助粗筛） | 高（建模首选） |

---

#### ElasticNet（L1 + L2 混合）

ElasticNet 同时加入 L1 和 L2 惩罚，通过混合比例 r 在两者之间取得平衡：

```
J(w) = Loss + λ × [ r × Σ|w_j| + (1 - r) × Σw_j² ]

  r = 1：退化为纯 L1（Lasso）
  r = 0：退化为纯 L2（Ridge）
  0 < r < 1：兼顾稀疏化与系数稳定性
```

当特征数量极多且存在高度共线特征群时，ElasticNet 能在自动筛选变量的同时保留共线群内的多个特征，优于纯 L1。

---

#### sklearn 中的 C 参数

sklearn 将正则化强度参数改写为 **C = 1/λ**，目标函数等价改写为：

```
J(w) = C × Loss + 惩罚项

  C 大（如 C=10） → λ 小 → 正则化弱 → 模型复杂，倾向过拟合
  C 小（如 C=0.01）→ λ 大 → 正则化强 → 模型简单，倾向欠拟合
```

**C 的选择方法——交叉验证网格搜索**：

```python
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import GridSearchCV

param_grid = {'C': [0.001, 0.01, 0.1, 1, 10]}

grid_search = GridSearchCV(
    LogisticRegression(penalty='l2', class_weight='balanced', max_iter=1000),
    param_grid,
    cv=5,
    scoring='roc_auc'
)
grid_search.fit(X_train_woe, y_train)

print(f"最优 C: {grid_search.best_params_['C']}")
print(f"对应 AUC: {grid_search.best_score_:.4f}")
```

**经验参考**：信贷评分卡场景中，经 WOE 编码后特征量纲已统一，C 通常在 0.01–1 之间，过大容易过拟合，过小模型区分度不足。

### 样本不平衡处理

信贷数据中好坏比例严重不平衡（如好:坏 = 20:1），直接训练会导致模型偏向预测所有人为好客户。

**常用处理方法**：

| 方法 | 做法 | 适用场景 |
|------|------|---------|
| class_weight='balanced' | sklearn 自动调整类别权重 | 简单快捷，首选 |
| 欠采样（Under-sampling） | 随机减少好客户样本 | 数据量充足时 |
| 过采样（SMOTE） | 合成新的坏客户样本 | 坏样本极少时 |

### 代码示例（sklearn）

```python
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler

# 标准化（逻辑回归对特征量纲敏感）
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)

# 训练模型
model = LogisticRegression(
    C=0.1,                    # 正则化强度（越小越正则）
    penalty='l2',             # L2 正则
    class_weight='balanced',  # 处理样本不平衡
    max_iter=1000,
    random_state=42
)
model.fit(X_train_scaled, y_train)

# 输出违约概率
prob = model.predict_proba(X_test_scaled)[:, 1]
```

## 五、模型评估

### 核心指标：KS 和 AUC

**KS（Kolmogorov-Smirnov）**：

KS 是风控建模中最常用的区分度指标，衡量模型把好坏客户分开的能力。

```
KS = max(好客户累积占比 - 坏客户累积占比)
```

| KS 值 | 评价 |
|-------|-----|
| < 0.2 | 较差，模型不可用 |
| 0.2 – 0.3 | 一般 |
| 0.3 – 0.5 | 良好，达到上线标准 |
| > 0.5 | 很好 |

**AUC（Area Under ROC Curve）**：

AUC 衡量模型对任意一对好坏客户的排序能力。

| AUC 值 | 评价 |
|--------|-----|
| 0.5 | 随机猜测，无效模型 |
| 0.6 – 0.7 | 一般 |
| 0.7 – 0.8 | 良好 |
| > 0.8 | 优秀 |

**KS 和 AUC 的关系**：通常 KS ≈ 2 × (AUC - 0.5)（近似关系）。

### PSI：模型稳定性监控

**PSI（Population Stability Index，群体稳定性指数）** 监控模型上线后评分分布是否发生偏移：

```
PSI = Σ (实际占比 - 预期占比) × ln(实际占比 / 预期占比)
```

| PSI 值 | 评价 |
|--------|-----|
| < 0.1 | 分布稳定，模型有效 |
| 0.1 – 0.25 | 轻微偏移，需关注 |
| > 0.25 | 显著偏移，模型可能需要重训 |

## 七、实战建议

1. **特征工程先于调参**：逻辑回归的瓶颈通常在特征质量，WOE 分箱做得好，KS 提升 5–10% 不是问题

2. **不要跳过单变量分析**：上模型前用 IV 值做特征筛选，去掉 IV < 0.02 的弱特征

3. **注意共线性**：两个高度相关的特征同时入模会导致系数不稳定，用 VIF（方差膨胀因子）检查

4. **时间窗口很关键**：训练集和测试集要按时间切分（不能随机），防止未来信息泄露

5. **监控不能少**：上线后每月计算 PSI，超过 0.25 就要重新评估模型有效性
