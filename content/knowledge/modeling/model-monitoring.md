## 一、为什么模型会失效？

模型在历史数据上训练，假设未来的数据分布与历史相似。但现实世界在不断变化：

- **客群变化**：获客渠道调整，新进来的用户与历史用户画像不同
- **产品变化**：额度上调/下调，改变了用户行为
- **宏观变化**：经济下行，整体信用风险上升
- **策略变化**：准入标准收紧/放松，改变了在贷客群结构

当这些变化发生时，模型在训练数据上学到的规律可能在当前数据上不再适用。**模型开始失效的典型表现**：
- 高分用户的实际坏账率不再比低分用户明显更低
- 模型的 KS 值开始持续下降
- 准入策略的通过率正常，但坏账率异常

**模型监控的目标**：及时发现这些变化，在模型完全失效（造成损失）之前做出响应。

## 二、PSI：人群稳定性监控

PSI（Population Stability Index）的公式定义与计算代码详见《风控建模核心评价指标》篇。这里聚焦监控运营层面。

### 监控基准的确定

PSI 需要一个**基准分布（Expected）**。实践中有两种选择：

| 基准选择 | 适用场景 | 注意 |
|----------|----------|------|
| 训练集评分分布 | 模型首次上线后监控 | 最常用，反映"建模时的客群" |
| 上月评分分布 | 监控短期变化趋势 | 能及时发现突变，但不反映与建模基准的偏离 |

建议同时维护两个基准：训练集基准（发现长期漂移）+ 上期基准（发现短期异动）。

### PSI 的判断标准与行动

| PSI 值 | 解读 | 行动建议 |
|--------|------|----------|
| < 0.1 | 稳定，分布变化可忽略 | 正常监控 |
| 0.1 – 0.2 | 轻微漂移，需关注 | 分析原因，提高监控频率 |
| 0.2 – 0.25 | 明显漂移，警告 | 深入分析，考虑模型优化 |
| > 0.25 | 严重漂移 | 立即评估是否需要重建模型 |

> **注意**：PSI > 0.2 是触发模型审查的警戒线，但不一定立即重建。需要同时查看 KS 值是否也在下降。

## 三、KS 衰减监控

KS 值的公式定义详见《风控建模核心评价指标》篇。这里重点讲上线后的持续追踪。

### 离线 KS vs 在线 KS

**离线 KS**：在训练集/测试集上计算，反映模型的理论区分能力。  
**在线 KS**：在真实部署数据上计算（需等表现期成熟），反映模型实际表现。

模型上线后，在线 KS 通常低于离线 KS，且会随时间缓慢下降。这是正常的。**快速大幅下降**才是预警信号。

### KS 衰减趋势图

每月（或每季度）计算在线 KS，绘制趋势图：

```
KS值
 ↑
0.45 ─────●────●
0.40           ──●──●
0.35                 ──●──●
0.30                       ──● ← 预警！
0.25
     M1  M3  M6  M9  M12  M18
         (模型上线月份)
```

**KS 衰减的判断标准**：

| 衰减幅度 | 建议 |
|----------|------|
| < 15% | 正常衰减，继续监控 |
| 15% – 30% | 明显衰减，分析原因，考虑特征更新 |
| > 30% | 严重衰减，启动模型重建流程 |

> 同时看绝对值：KS 从 0.50 降到 0.35（下降 30%）比从 0.35 降到 0.24（下降 31%）情况更严重，因为后者已接近无效模型。

## 四、评分漂移分析

除了 PSI 和 KS，还需要追踪**评分与实际坏账率的对应关系**是否发生变化。

### 评分-坏账率曲线监控

理想状态：高分段坏账率低，低分段坏账率高，这条曲线随时间保持稳定形态。

监控方法：按评分分箱，分别统计各箱在表现期内的实际坏账率，与训练时的预期坏账率对比：

| 分箱 | 训练时预期坏账率 | 当前实际坏账率 | 偏差 |
|------|---------|---------|------|
| 低分（< 580） | 25% | 32% | +7% ↑ |
| 中低分（580–620） | 15% | 18% | +3% ↑ |
| 中高分（620–660） | 8% | 8.5% | +0.5% → |
| 高分（> 660） | 3% | 2.8% | -0.2% → |

上表说明：低分段的实际坏账率明显高于预期，模型在低分段低估了风险。这可能意味着：
- 低分段的特征已经失效（某个特征不再预测信用风险）
- 低分段的客群结构发生了变化
- 需要对低分段的准入策略收紧，或局部重新训练

### 特征漂移分析（CSI）

当模型整体出现问题时，需要下钻到特征层面，找到具体哪个特征发生了漂移。对每个入模特征分别计算特征 PSI（即 CSI，特征稳定性指数）：

| 特征 | CSI | 状态 |
|------|-----|------|
| 近3个月逾期次数 | 0.08 | 稳定 |
| 征信查询次数 | 0.31 | 严重漂移！ |
| 月收入 | 0.15 | 轻微漂移 |
| 账户利用率 | 0.11 | 轻微漂移 |

征信查询次数 CSI 严重漂移，说明当前客群的征信查询行为模式已与训练时显著不同（可能是获客渠道变化，或征信政策变化影响了查询记录）。

```python
# 批量计算所有入模特征的 CSI
from evaluation_utils import calc_psi   # PSI/CSI 公式相同，详见评价指标篇

for feat in model_features:
    csi = calc_psi(df_train[feat].values, df_current[feat].values)
    status = '严重漂移！' if csi > 0.25 else ('轻微漂移' if csi > 0.1 else '稳定')
    print(f"{feat:20s}  CSI={csi:.4f}  {status}")
```

## 五、监控体系的整体框架

### 监控频率设定

| 监控类型 | 建议频率 | 说明 |
|----------|----------|------|
| 评分分布（PSI） | 月度 | 及时发现客群漂移 |
| KS 值 | 季度 | 表现期需要足够长 |
| 评分-坏账率对应关系 | 季度 | 需要观察完整表现期 |
| 特征漂移（CSI） | 月度 | 辅助定位问题根源 |
| 模型整体效果综合评审 | 半年度 | 综合判断是否需要重建 |

### 自动化预警流程

```
每月数据更新
    │
    ├─ 计算评分分布 PSI（与训练集基准对比）
    │     IF PSI > 0.2 → 发送预警，触发模型审查
    │
    ├─ 计算各特征 CSI
    │     IF 任一特征 CSI > 0.25 → 标记该特征，排查上游数据源
    │
    └─ 每季度：计算在线 KS
          IF KS 衰减 > 15% → 启动模型评估流程
```

## 六、模型重建的决策框架

发现预警信号后，不是立刻重建，而是按步骤评估：

### 第一步：判断漂移的性质

| 漂移类型 | 特征 | 应对方案 |
|----------|------|----------|
| 客群结构变化 | PSI 高，但各特征 CSI 均低 | 重新采样训练数据，重建模型 |
| 特定特征失效 | 某个特征 CSI 高 | 替换该特征，局部优化 |
| 上游数据口径变更 | 特征分布突变而非渐变 | 修复数据，评估是否需要重训 |
| 宏观风险整体上升 | 所有评分段坏账率均升 | 整体下调通过阈值，研究是否重建 |

### 第二步：评估当前模型残余价值

- 当前在线 KS 还有多少？（KS > 0.35 说明仍有区分度）
- 哪些分数段受影响最大？（局部问题可通过调整策略弥补）
- 距上次重建已多久？（建模数据窗口是否需要向后滚动）

### 第三步：决定行动

| 情况 | 推荐行动 |
|------|----------|
| PSI < 0.2 且 KS 衰减 < 15% | 继续监控，无需操作 |
| PSI 0.2–0.25 或 KS 衰减 15–30% | 启动模型优化（特征更新 / 样本窗口滚动） |
| PSI > 0.25 或 KS 衰减 > 30% | 立即启动模型重建 |
| KS < 0.2（模型几乎无效） | 紧急重建，临时用规则兜底 |

## 七、常见误区

**误区1：PSI 高就一定要重建模型**

PSI 高说明分布变化大，但如果 KS 值和实际表现没有明显恶化，模型仍然有效。PSI 是预警信号，不是直接的"重建命令"。

**误区2：模型越新越好**

用最近 3 个月数据重建的模型，样本量不足、表现期不完整，反而不稳定。重建模型需要足够的样本量和足够长的表现期（参见 OOT 验证篇）。

**误区3：只监控整体，不监控分群**

整体 KS 稳定，可能掩盖了某个重要子群（如新户、某个渠道）的模型严重失效。必须分渠道、分产品分别监控。

**误区4：特征 CSI 异常即换特征**

特征分布突变首先应排查上游数据源是否发生字段口径变更，而非直接删除特征——口径问题修复后特征可能仍然有效。

## 小结

| 监控工具 | 监控对象 | 预警阈值 |
|----------|----------|----------|
| PSI | 评分分布稳定性 | > 0.2 预警 |
| KS 衰减 | 模型区分能力 | 下降 > 15% 预警 |
| 评分-坏账率曲线 | 模型预测准确性 | 系统性偏差 |
| CSI（特征 PSI） | 单特征分布稳定性 | > 0.25 需查明原因 |

模型不是"一次建好、一劳永逸"的工具，而是需要持续维护的业务资产。监控体系建立后，还需要配合 Vintage 分析（见 OOT 验证篇）周期性评估各期放款质量，形成完整的模型生命周期管理闭环。
