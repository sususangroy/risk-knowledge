## 业务背景

风控模型体系由反欺诈模型、申请评分卡（A卡）、行为评分卡（B卡）、催收评分卡（C卡）构成。C卡在实践中长期被忽视，原因有二：一是贷前风控是重中之重；二是 M1+ 阶段普遍委外处理。随着委外被逐步淘汰，自建催收团队的精细化运营需求推动了 C卡的重要性上升。

**核心概念**

- **入催**：用户当期逾期，进入催收库
- **出催**：用户结清逾期账单，离开催收库
- **催收目标**：降低入催率 + 提高出催率

---

## Part 1. 模型分类

贷后催收按逾期天数分为四个阶段：

| 阶段 | 逾期天数 | 说明 |
|------|----------|------|
| M0 | DPD 1–30 | 出催概率最高，是催收重点 |
| M1 | DPD 31–60 | 出催概率明显下降 |
| M2 | DPD 61–90 | 难催，损失加重 |
| M3 | DPD 91–120 | 损失为所有剩余未还金额 |

### M0 阶段模型

M0 阶段出催概率最高，可细分为两类模型：

**缓催响应模型**

部分用户仅因遗忘而逾期，不催或弱催（短信、邮件）即可还款。缓催响应模型预测哪些用户适合缓催，节省人力。缓催响应时间需根据该时间段出催率变化来确定。

**贷后 N 天流转模型**

去除缓催人群后，预测剩余入催用户在第 N 天至 M0 结束前的出催概率。构建思路：

1. 以第 N 天为观察点（N 通过出催率曲线分析确定，通常大部分易催用户在前几天已出催）
2. 第 N 天后仍未出催的样本作为训练样本
3. 去除缓催样本影响，让模型更专注于难催用户

> **注意**：若直接构建整体 M0 模型，缓催期内出催用户占多数时，模型实质等同于缓催响应模型，剔除缓催样本后效果大打折扣。

### M1+ 阶段模型

M1+ 用户出催率极低，但损失为所有剩余未还金额，因此需要模型精准识别出催概率较高的用户，提升催收效率。

- 初期可直接构建 M1+ 整体模型（样本窗口 DPD 31–120）
- 样本充足后，分别构建 M1、M2、M3 精细化模型
- Target 定义：M1+ 用户**出催一期**即视为出催（否则正样本过少）

## Part 2. 样本选择与分群

### 建模样本窗口选择

确定窗口期需综合考量三个维度：

| 维度 | 说明 |
|------|------|
| 特征覆盖度 | 保证不同月份数据厚薄度一致 |
| 出催动作变化 | 催收动作大幅变动会影响出催率，需保持稳定 |
| 客群变化 | 客群结构变化影响入催样本分布 |

### 观察点与表现期

| 模型类型 | 观察点 | 表现期 |
|----------|--------|--------|
| 缓催响应模型 | 还款日 | 前 N 天 |
| 贷后 N 天流转模型 | 第 N 天 | 第 N 天至 M0 结束 |
| M1+ 模型 | M1 起始日 | 相应阶段结束 |

### 合案处理

同一用户可能同时有多笔在途订单同时入催，需考虑是否合案：

- **不合案**：多笔订单视为独立样本，导致不出催样本偏多
- **合案**：同一用户相近观察点入催订单合并至前一观察点

> M1+ 用户建议**必须合案**，否则正样本（出催样本）严重不足。

### Target 定义

- 合案样本：表现期内所有到期账单全部还清 → 出催（部分还清可定义为灰样本）
- 未合案样本：当期逾期账单在表现期内还清 → 出催

### 样本分群思路

分群目标：**不同群体间出催率有显著差异，或可用特征存在明显差异**

| 分群维度 | 示例 |
|----------|------|
| 新老入催 | 首次入催 vs 再次入催（后者有催记信息） |
| MOB 信息 | 还款月份数、催记月份数（数据厚薄不同） |
| 订单详情 | 利率、期限、贷款金额 |

分群粒度越细，模型效果通常越好，但需结合团队人力和时间综合决定。

## Part 3. 特征选择

催收模型特征分为三大类：

### 还款能力特征

衡量用户是否具备还款的经济条件：

- **收入**：自填收入、第三方收入核验数据
- **负债**：内部负债、外部多头借贷信息
- **学历**：自填学历、外部学历验证数据

### 还款意愿特征

衡量用户是否主观愿意还款：

- **申贷行为**：历史申贷记录、贷前贷后申贷行为变化
- **履约行为**：历史还款记录、逾期记录
- **催记行为**：催收接触记录、接触后反应

### 基本信息特征

- A 卡分（贷前信用评分）
- B 卡分（贷中行为评分）
- 用户基本信息（年龄、性别等人口统计特征）

> 将特征归类为**能力类**和**意愿类**，可在模型评分基础上分别计算还款能力分和还款意愿分，为后续四象限分析奠定基础。

---

## Part 4. 模型选择

**推荐使用评分卡（LR）模型**，原因如下：

1. **可拆分性**：可将能力类特征得分与意愿类特征得分分别求和，直接得到还款能力分和还款意愿分
2. **可解释性**：便于归因，向业务方解释用户高分/低分的原因
3. **便于策略制定**：评分卡分数与策略阈值直接挂钩，操作便捷

对于分群建模的情况，需先进行**分数校准**，确保各群体的评分在同一尺度上：

1. 对整体样本分数做等频分箱，得到分箱阈值
2. 对每个群体按上述阈值分箱
3. 计算各群体每箱的均分（AvgScore）和坏样本率（BadRate）
4. 绘制 AvgScore vs BadRate 曲线，各群体曲线拟合度高则说明分数已校准

## 总结

C 卡建模的核心难点在于**样本处理**（合案、分群、观察点选择）而非算法本身。催收模型交付给业务只是开始，后续基于模型的**能力意愿分析**和**敏感性分析**才是催收策略落地的关键。
